# Trade MVP 立项条件清单

日期：2026-03-30

## 目标定义

Trade MVP 的目标不是扩充装饰或空间建设内容，而是补上 V1 中期经济闭环缺口，验证以下链路成立：

`基础资源溢出 -> 自动贸易转金币 -> 金币投入招募/装备 -> 生产与战备增强 -> 更高阶段 Boss`

Trade 系统在 V1 中应被定义为“主循环增强器”，而不是新的独立玩法层。

## 立项前置条件

- 当前桌面 Demo 已具备 V1 主循环基本可玩形态，包括：全局策略、角色优先级、自动工作、资源增长、建筑建造、科技推进、招募链路、Boss 刷新与掉落、存档读档。
- 当前版本不存在阻塞 30 到 45 分钟连续试玩的 P0 级问题。
- 产品文档完成一次范围同步，明确 Trade 是否正式并入 V1。
- 若贸易站并入 V1，需同步修订“V1 核心建筑只有九类”的权威口径，避免 [README.md](/D:/GameDesign/Projects/idle-collective/docs/v1/README.md) 、[GAME_DESIGN_DOCUMENT.md](/D:/GameDesign/Projects/idle-collective/docs/v1/核心设计/GAME_DESIGN_DOCUMENT.md) 与 [TRADE_SYSTEM_DESIGN.md](/D:/GameDesign/Projects/idle-collective/docs/v1/系统设计/TRADE_SYSTEM_DESIGN.md) 冲突。
- 金币消耗端至少有两项在 MVP 内真实可用且可感知：招募候选成员、手动刷新招募站、抽取或购买装备。
- Trade 方案边界已冻结：唯一正式交易入口为贸易站；仅处理木材、石材、食物；采用盈余交易；禁止卖空；不做动态价格。
- 贸易站最小 UI 范围已确认，包括：开关、偏好、保留库存、预计收益、每分钟金币产出、交易反馈。
- 建筑队列缓冲规则已确认，避免“刚攒够建材又被系统卖掉”。
- QA 已接入立项评审，当前未完成的试玩检查项需纳入 Gate 1 追踪。

## Gate 1 验收标准

- 已明确 Trade 要解决的是“资源输出口不足”，不是“资源消耗入口不足”。
- 已确认 Trade 不替代建造、科技、Boss 主循环，只为其提供经济反馈。
- 已确认 Trade 的唯一入口为贸易站，不引入商人事件、市场、订单板等平行系统。
- 已确认可交易资源仅为木材、石材、食物三类基础资源。
- 已确认核心零件、装备、角色不可交易。
- 已确认金币职责收束为招募与装备，不承担主建筑升级货币职责。
- 已确认自动交易默认按“盈余”运行，且保留库存可手动配置。
- 已确认建造/升级排队资源有缓冲保护，不会被交易系统误卖。
- 已确认 Trade 与全局策略的关系是“推荐偏好”，不是“强制覆盖玩家设置”。
- 已完成一轮纸面数值校验，证明 Trade 不会成为前期开局最优解，也不会让中期金币失控。
- 已输出完整测试场景，覆盖解锁、建造、自动交易、停卖保护、金币消耗、存档读档。

## 放行条件

- 所有前置条件已满足，且无未决的范围冲突。
- 权威文档中的 V1 范围口径一致。
- 与主循环相关的阻塞试玩项已关闭，或已明确降级为非阻塞问题。
- 已完成至少一次 30 到 45 分钟人工试玩验收，并形成问题单。
- 已验证“资源转金币”与“金币转招募/装备”两端都能跑通，不存在单边成立。
- 已验证自动交易不会破坏生存稳态，不会卖空食物，不会挤占关键建材。
- 已验证 Trade UI 文案、提示、反馈与现有系统命名统一。
- 无 P0 未解决问题；P1 问题如保留，必须有负责人和修复日期。
- 已确认 MVP 不夹带 V1.2 内容，不以立项名义扩 Scope。

## MVP 范围

### 系统范围

- 新增核心建筑：贸易站。
- 贸易站作为 V1 唯一正式资源交易入口。
- 贸易站自动将盈余木材、石材、食物转换为金币。
- Trade 接入现有资源系统、建筑系统、招募系统、装备系统、全局策略系统。
- 金币继续保留为 V1 核心资源之一，但在中期承担明确规划货币职责。

### 规则范围

- 解锁前置：仓库已建成，且生产科技 1 级已解锁。
- 建造成本：木材 120、石材 60、金币 150。
- 默认保留库存：木材 120、石材 80、食物 100。
- 交易公式采用“当前库存 - 保留库存 - 排队缓冲”。
- 自动交易支持开关。
- 支持 4 种贸易偏好：均衡贸易、木材优先、石材优先、食物保守。
- 贸易站支持 1 到 3 级成长，只提升成交上限、效率和保留库存调节空间。
- 基础价格采用静态价格。
- 食物价格低于木材和石材，避免鼓励卖粮破坏生存。

### UI 范围

- 贸易站面板展示当前等级。
- 贸易站面板展示当前贸易偏好。
- 贸易站面板展示三类资源库存。
- 贸易站面板展示三类资源保留线。
- 贸易站面板展示预计下轮金币收入。
- 贸易站面板展示当前每分钟金币产出。
- 地图或浮字明确反馈本轮卖出了什么、获得了多少金币。
- 当资源首次明显溢出时，给出贸易站引导提示。

### 验证范围

- 验证新档可正常解锁并建造贸易站。
- 验证自动交易、关闭交易、切换偏好、调整保留库存均可生效。
- 验证金币能稳定流向招募和装备。
- 验证存档读档后贸易状态一致。
- 验证 8 到 12 人规模下不会破坏现有主循环节奏。

## 明确不做内容

- 地板
- 墙体
- 道路
- 围栏
- 装饰与空间美化
- 商人事件
- 市场价格实时波动
- 多货币体系
- 奢侈品与高阶商品加工链
- 手动高频点击卖货玩法
- 核心零件交易
- 装备交易
- 角色交易
- 金币反向兑换基础资源
- 独立贸易职业、商队 AI、运输链模拟

## 主要风险

| 风险 | 影响 | 说明 | 应对 |
|------|------|------|------|
| 文档口径冲突 | 高 | 当前 V1 权威文档写明核心建筑 9 类，贸易站纳入后将直接冲突 | 立项前同步修订权威文档 |
| 金币价值不足 | 高 | 若招募或装备端消耗弱，Trade 只能制造“无意义金币” | 立项前确认至少两条金币消耗链真实可用 |
| 自动交易破坏生存 | 高 | 食物或关键建材被误卖，会直接伤害挂机稳定性 | 强制保留库存、排队缓冲、食物保守策略 |
| 抢走主循环地位 | 高 | 若 Trade 收益过强，会压过建造、科技、Boss 的成长地位 | 用静态价格、成交上限、升级节奏限制收益 |
| 反馈不足导致无感 | 中 | 贸易是后端系统，若缺少 UI 反馈，玩家会感受不到价值 | 将浮字、面板收益、引导提示列为 MVP 必做 |
| 范围蔓延 | 中 | 容易从 Trade 扩到墙、地板、商人、市场系统 | 在立项文件中锁死“不做内容” |
| QA 覆盖不足 | 中 | 当前试玩检查项大量未勾完，存在带病立项风险 | Gate 1 前补一次完整试玩并形成问题单 |
| 数值失衡 | 中 | 价格、保留线、成交上限不当会导致前中期节奏异常 | 先锁首版安全区间，再做人工试玩调参 |

## 验收清单

### 功能验收

- [ ] 新档可正常进入主界面并完成基础开局。
- [ ] 满足前置条件后可建造贸易站。
- [ ] 贸易站是唯一正式交易入口。
- [ ] 贸易站只交易木材、石材、食物。
- [ ] 自动交易开启后，系统只出售盈余资源。
- [ ] 当前库存小于等于保留库存时不会出售对应资源。
- [ ] 建造/升级排队中的关键资源会被缓冲保护。
- [ ] 关闭自动交易后，贸易站暂停工作。
- [ ] 切换贸易偏好后，成交资源倾向发生变化。
- [ ] 切换全局策略时，只给出贸易建议，不覆盖玩家已设置参数。
- [ ] 金币可用于招募候选成员、手动刷新或装备获取。
- [ ] 存档后读档，贸易站开关、偏好、保留库存、金币产出状态一致。

### 体验验收

- [ ] 玩家能通过提示和反馈理解“贸易站把盈余资源转为金币”。
- [ ] 地图或浮字能明确显示卖出资源与获得金币。
- [ ] 贸易站面板能看懂当前收益与当前配置。
- [ ] 玩家不会因贸易系统产生“为什么资源突然少了”的困惑。
- [ ] 30 到 45 分钟试玩过程中，Trade 能被感知为中期稳定输出口。

### 平衡验收

- [ ] 贸易站不会成为最优开局建筑。
- [ ] Trade 不会让建筑升级和 Boss 突破失去门槛意义。
- [ ] 食物交易不会稳定破坏生存稳态。
- [ ] 金币产出不会在中期快速失控。
- [ ] 金币不会长期堆积而缺乏用途。

### 范围验收

- [ ] 未引入市场价格波动。
- [ ] 未引入商人事件。
- [ ] 未引入多货币体系。
- [ ] 未引入地板、墙体、道路、围栏。
- [ ] 未引入高阶商品加工链。
- [ ] 未引入 Trade 相关额外职业或运输模拟。

## 结论口径

满足以上前置条件、Gate 1 验收标准与放行条件后，Trade MVP 才应正式立项。  
若“权威文档冲突未解决”或“金币消耗端不成立”，则本项应判定为“暂不放行”，先修正基础口径与依赖闭环。
