# Demo QA Checklist

## 启动

- [x] Windows 桌面包可以正常启动
- [ ] 新档可以进入主界面
- [ ] 试玩说明清晰可读

## 核心循环

- [ ] 可以设置全局策略
- [ ] 可以切换角色优先级
- [ ] 启用全局策略后，角色级优先级不可手动编辑
- [ ] 关闭全局策略后，角色级优先级恢复可编辑
- [ ] 角色会自动执行工作
- [ ] 资源会稳定增长或消耗

## 建筑与聚落

- [ ] 建筑会消耗资源并产生功能效果
- [ ] 聚落宜居度会影响需求恶化、工作效率和招募质量
- [ ] 聚落发展度会影响 Boss 难度

## 招募

- [ ] 招募站存在并可查看候选成员
- [ ] 商队会按周期刷新候选成员
- [ ] 候选质量与宜居度、发展度有关
- [ ] 招募购买消耗金币
- [ ] 刷新候选消耗金币

## 科技与 Boss

- [ ] 科技可以推进
- [ ] Boss 会定时刷新
- [ ] Boss 可以挑战
- [ ] Boss 会掉落核心零件
- [ ] 核心零件可用于升级或关键突破

## 贸易站与金币经济 (Sprint 2/3)

- [ ] 贸易站建筑可建造
- [ ] 贸易站完工后可开启/关闭自动贸易
- [ ] 贸易只会卖出木材、石材、食物的盈余
- [ ] 贸易不会卖空保留库存
- [ ] 最近成交结果可查看
- [ ] 估算收益可查看
- [ ] 金币能流向招募购买
- [ ] 金币能流向招募刷新
- [ ] 金币能流向装备购买
- [ ] 金币不会在前中期失控堆积
- [ ] Trade 不是唯一最优解

## 发展度反馈 (Sprint 3)

- [ ] 发展度四项来源可查看（研究/建筑/核心零件/战备）
- [ ] 发展度来源数值与公式一致
- [ ] 玩家能理解发展度来源和作用

## 性能与交互 (Sprint 3)

- [ ] 地图拖动不卡顿到影响操作判断
- [ ] 底部面板收起/展开联动正常
- [ ] 关键状态反馈清晰

## 存档

- [ ] 可以存档
- [ ] 可以读档
- [ ] 读档后 Trade 状态一致（开关、保留库存、最近结果）
- [ ] 读档后发展度状态一致
- [ ] 读档后招募状态一致
- [ ] 读档后金币状态一致

## 自动化测试

- [ ] `npm run typecheck` 通过
- [ ] `npm run test:run` 通过
- [ ] `npm run build` 通过
- [ ] `npm run package` 通过

## 已知问题

- [ ] 新问题已经记录到问题单
- [ ] 阻塞试玩的问题有明确负责人
- [ ] P0 问题已分配或降级
- [ ] P1 问题有修复时间点或降级理由

## 退出条件

- [ ] 至少完成 1 次完整 30-45 分钟试玩并沉淀结果
- [ ] 无未分配负责人的 P0
- [ ] Demo checklist 完整更新
- [ ] 满足 V1 Demo 里程碑退出条件
