# Sprint 1

## 目标

在 2 周内完成一个可对外试玩的 Windows 桌面 Demo，验证 V1 的核心闭环成立：

`设定优先级或策略 -> 聚落自动执行 -> 获取资源与招募成员 -> 推进研究和建造 -> 挑战定时 Boss -> 获得核心零件 -> 升级并进入下一轮规划`

## 时间范围

- 开始：2026-03-28
- 结束：2026-04-10

## 本轮交付

- Windows 桌面试玩包
- 仓库内 V1 权威文档
- 试玩说明
- Demo QA 清单与已知问题记录

## 工作拆分

| 方向 | 任务 | 结果 |
| --- | --- | --- |
| 生产 | 建立 Sprint、里程碑和 QA 资产 | 形成可追踪版本基线 |
| 文档 | 同步 `docs/v1` 权威策划 | 仓库内可直接引用 |
| 文档 | 补齐试玩说明和验收文档 | 可试玩、可回顾 |
| 程序 | 打通 V1 主循环 | 形成可玩闭环 |
| 程序 | 完成全局策略与角色优先级互锁 | 自动执行逻辑成立 |
| 程序 | 接通宜居度和发展度 | 影响核心系统 |
| 程序 | 完成招募站和商队刷新 | 可稳定招募新成员 |
| 程序 | 完成 Boss 与核心零件回流 | 推动阶段升级 |
| 程序 | 保证可存档和可读档 | Demo 可持续体验 |
| UI | 明确资源、需求和策略反馈 | 外部试玩者可理解 |
| QA | 冒烟与回归验证 | 形成试玩保障 |

## 验收标准

- 新档进入后 10 分钟内能理解主循环。
- 30 到 45 分钟内能完成第一次有效突破。
- 不依赖开发者讲解，也能完成一次“规划 -> 执行 -> Boss -> 升级”的闭环体验。
- 策略切换、资源变化、Boss 奖励和存档恢复都能稳定展示。

## 风险

1. 旧实现与 V1 文档仍可能存在偏差，需要持续对齐。
2. 构建产物和发布目录较多，必须保持版本基线清晰。
3. 若招募、科技、Boss 或存档链路出现断裂，Demo 节奏会被拉长。

## 备注

- 本轮只做闭环，不做大规模内容扩充。
- 可以接受临时美术和简化演出，但不能缺少核心反馈。
