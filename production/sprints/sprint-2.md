# Sprint 2

## 目标

在 1 周内完成 Trade MVP、金币经济闭环与发展度公式重构，验证中期资源输出口是否稳定成立：

`盈余资源 -> 贸易站自动换金币 -> 招募 / 刷新 / 装备消耗金币 -> 更强生产与战斗 -> 更高阶段推进`

## 时间范围

- 开始：2026-03-30
- 结束：2026-04-05

## 本轮交付

- [系统策划文档格式规范](/D:/GameDesign/Projects/idle-collective/docs/系统策划文档格式规范.md)
- 贸易站建筑与最小贸易面板
- 金币 Source/Sink 闭环
- 新版聚落发展度公式
- 回归测试与桌面包构建产物

## 工作拆分

| 方向 | 任务 | 结果 |

## 验收标准

- 贸易站完工后可开启/关闭自动贸易。
- 贸易只会卖出木材、石材、食物的盈余，不会卖空保留库存。
- 金币能稳定流向招募购买、招募刷新与装备购买。
- 发展度不会被单一建筑等级直接刷满。
- 存档读档后保留 trade 开关、保留库存和最近成交结果。
- `npm run typecheck`、`npm run test:run`、`npm run build`、`npm run package` 全部通过。

## 风险

1. 贸易站上线后可能暴露新的金币通胀问题，仍需试玩调参。
2. 当前建造成本为放置即扣，因此“建造预留”先按最小模型处理；若后续恢复真实建造队列，需要复查 Trade 逻辑。
3. 贸易站本轮不开放升级，后续若开放，需要重新校准发展度与经济节奏。

## 备注

- 本轮只做 Trade MVP，不做完整市场系统。
- 保留初始金币，贸易站主要承担中期稳定金币来源。
