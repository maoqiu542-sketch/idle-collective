## V1 核心玩法一句话

玩家设定角色技能优先级和全局策略预设，AI 系统根据优先级自动调度角色执行采集、建造、研究、战备等任务，推动聚落从初创→发展→繁荣的自然演进。

## V1 范围（固定版）

| 维度 | V1 范围 |
|------|---------|
| 角色数量 | 最多 12 人 |
| 核心资源 | 木材、石材、食物、金币、核心零件 |
| 核心建筑 | 房屋 / 厨房 / 农场 / 伐木机 / 采石机 / 研究台 / 招募站 / 仓库 / 兵营（共 9 类） |
| 科技分支 | 生存 / 生产 / 战备（共 3 条） |
| Boss | 定时 30 分钟刷新，难度随聚落发展度分层 |
| 策略预设 | 生存稳态 / 扩张建造 / 科研冲刺 / 备战 Boss（共 4 套） |
| 角色控制 | 技能优先级 1-10 级，可禁用/强制任务类型 |
| 装备 | 辅助型六维装备，提升战斗属性 |
| 贸易 | 自动出售木材/石材/食物盈余，换取金币 |
| 存档 | 本地存档，版本校验 |

## 核心循环

```
玩家设定优先级/策略预设
  → AI 系统自动分配角色
  → 角色执行任务（采集、建造、研究、战备等）
  → 资源积累、建筑建设、研究推进
  → Boss 定时刷新，击败后获得核心零件
  → 核心零件用于建筑升级和存储扩容
  → 发展度提升，解锁更高难度内容
```

## 设计原则

1. **玩家做规划，角色自动执行** - 玩家决定"做什么优先级高"，角色决定"怎么做"
2. **优先级驱动，而非队列驱动** - 角色自动根据优先级、状态和条件选择最优任务
3. **生存大于生产** - 生存类任务具备最高打断权，防止挂机崩盘
4. **策略预设是机制** - 切换预设直接改变角色行为，3-10 分钟内可见差异
5. **Boss 是阶段门槛** - Boss 是成长链的自然部分，击败后必须有新的成长选项
6. **行为差异可见** - 玩家所有操作必须在 UI 上有即时反馈

## 设计目标

1. 新玩家 10 分钟内理解"我设优先级，角色自动执行"的核心循环
2. 核心循环在 8-12 人规模下成立
3. 切换四套策略后 3-10 分钟内聚落行为出现明显差异
4. 玩家完成一个建筑或研究后可以看到明确的新内容
5. 5 类核心资源非消耗品时间平均（任何资源不会永远用不完或永远短缺）

## 系统依赖关系

```text
基础系统（无依赖）
├── 资源系统
├── 角色系统
│   └── 需求系统
│       └── 角色状态
├── 建筑系统
└── 地图系统

一级系统（依赖基础系统）
├── 科技系统（依赖建筑）
├── 招募系统（依赖建筑-招募站 + 角色）
│   └── 角色商店
├── 贸易系统（依赖建筑-贸易站 + 资源）
└── 装备系统（独立模块）

二级系统（依赖一级系统）
└── Boss 系统（依赖角色 + 科技 + 装备 + 发展度）

全局系统（贯穿所有）
├── AI 系统（依赖资源 + 角色 + 建筑）
├── PlayerInterventionManager（依赖角色 + AI）
├── 全局策略预设（依赖 AI + PlayerIntervention）
└── 存档系统（依赖所有数据持有层）
```
