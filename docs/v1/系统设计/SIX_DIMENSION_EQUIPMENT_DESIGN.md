# 六维装备与战斗属性设计

## 1. 设计目标

V1 保留战斗属性与装备系统，但不把装备做成主成长驱动。装备只负责辅助 Boss 战和阶段突破，让玩家可以感知成长，却不会被刷装循环绑架。

## 2. V1 保留范围

### 2.1 战斗属性

V1 仅保留四项战斗属性：

- `ATK`：攻击
- `DEF`：防御
- `HP`：生命值
- `SPD`：速度

这四项是 V1 唯一进入战斗计算的核心属性。

### 2.2 装备槽位

V1 只保留三类装备槽：

- 武器
- 护甲
- 饰品

不再拆分更多部位，也不引入细碎词条体系。

### 2.3 品质

V1 装备品质只保留四档：

- 普通
- 精良
- 稀有
- 史诗

不做传说、神话等更高阶扩展。

## 3. 基础公式

### 3.1 战斗力

```text
combat_power = ATK * 2 + DEF * 1.5 + HP * 0.1 + SPD * 20
```

该数值用于 Boss 难度匹配、结算提示和阶段反馈，不作为唯一战斗结果判定。

### 3.2 伤害

```text
damage = max(1, floor(ATK * (1 - DEF / (DEF + 100))))
```

该公式足够稳定、易理解，适合 V1 快速验证。

## 4. 装备来源

V1 装备主要来自：

- Boss 掉落
- 商队少量售卖
- 阶段奖励

装备不作为主线资源消耗的唯一出口。

## 5. 属性分配原则

- 武器偏向 `ATK`
- 护甲偏向 `DEF` 和 `HP`
- 饰品偏向 `SPD` 或少量综合属性

V1 不要求复杂词缀组合，只要求装备能看出“方向差异”。

## 6. 与主循环关系

- 装备提升 Boss 战效率
- Boss 掉落推动阶段升级
- 阶段升级反过来影响 Boss 难度与商队质量

这样形成闭环，但不会把装备变成独立刷装游戏。

## 7. 验收标准

- 玩家能在界面上清楚看到四项战斗属性。
- 装备槽仅为三类，不出现额外槽位。
- 装备不会影响资源生产主循环。
- 战斗结果与属性变化能稳定反映在 Boss 交互中。
