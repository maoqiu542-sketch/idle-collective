# Idle Collective V1 贸易系统设计文档

> **Status**: Proposal  
> **Author**: Codex  
> **Last Updated**: 2026-03-30  
> **Implements Pillar**: 让资源拥有稳定输出口，并把经济循环接回招募与装备成长

## Overview

当前 V1 的主要问题不是资源输入不足，而是资源在中期缺少稳定输出口。  
玩家在完成几座核心建筑后，木材、石材、食物会逐渐出现溢出，但这些资源很难继续转化为新的规划压力和成长反馈。

V1 贸易系统的目标是补上这条链路：

`资源生产 -> 资源交易 -> 金币积累 -> 招募 / 装备 -> 更强生产与战斗 -> 更高阶段 Boss`

本系统优先解决“资源有去处、金币有来源、招募与装备更有驱动力”三个问题。  
本系统 **不承担** 装饰、空间美化、墙体防御、地板铺设等表达型功能，这些内容后置到 V1 之后的扩展版本。

## Design Decision

### 为什么先做贸易系统，而不是先扩地板与墙

1. 当前问题是输出口不足，不是资源消耗入口不足。
2. 地板、墙、围栏更偏空间表达与后期资源沉淀，不是当前 V1 主循环最缺的一环。
3. 贸易系统可以直接增强已经存在的两条成长反馈：
   - 招募
   - 装备
4. 贸易系统能让金币从“少量辅助货币”升级为“中期规划货币”，让资源规划更有意义。

结论：

- V1 先补贸易系统。
- 地板、墙、道路、围栏作为 **V1.2 以后** 的空间建设层处理。

## Player Fantasy

玩家应当感受到：

- “我的聚落不只是囤资源，而是在把资源转成下一阶段成长。”
- “我做的是资源规划，而不是手动卖垃圾。”
- “我能决定保库存还是卖盈余，这会改变招募速度和战备速度。”

## Core Rules

### 1. 新增核心建筑：贸易站

贸易站是 V1 唯一正式的资源交易入口。

作用：

- 自动将 **盈余基础资源** 转换为金币
- 为招募与装备系统提供稳定金币来源
- 把中期资源溢出重新转成成长动力

### 2. 贸易站只处理三类基础资源

V1 可交易资源仅包括：

- 木材
- 石材
- 食物

以下资源 **不可交易**：

- 核心零件
- 装备
- 角色

### 3. 贸易站默认按“盈余”交易，不允许卖空

贸易系统不允许直接卖空资源。  
每类资源都必须保留一个最低库存线。

交易逻辑：

- 当前库存 `<= 保留库存`：不出售
- 当前库存 `> 保留库存`：只出售超出部分

### 4. 贸易站是自动运行建筑，不增加日常操作负担

玩家不需要频繁手动点击出售。  
玩家负责的是：

- 开启或关闭自动交易
- 选择交易偏好
- 设定每类资源的最低保留库存

角色负责的是：

- 自动生产资源
- 自动把盈余资源送入交易

### 5. 金币的主要输出端

V1 中，金币的核心用途明确收束为：

- 招募候选成员
- 手动刷新招募站
- 抽取或购买装备

金币不承担建筑升级主资源职责。  
建筑升级仍优先消耗基础资源与核心零件。

## Detailed Design

### Core Building

| 建筑 | 主功能 | 宜居度贡献 | 是否需要值班 |
|------|--------|-----------|--------------|
| 贸易站 | 自动把盈余木材/石材/食物转换为金币 | +2 | 否 |

### 建筑定位

贸易站属于 **生产后端建筑**，不是直接生产建筑。

它的作用不是产资源，而是：

- 吃掉中期溢出资源
- 把资源节奏转成金币节奏
- 让招募和装备真正形成稳定的经济回路

### 解锁条件

建议 V1 采用轻量解锁：

- 建造前置：仓库已建成
- 科技前置：生产科技 1 级

这样可以保证：

- 玩家先理解基础资源与建造
- 再理解“资源管理 -> 货币管理”的第二层规划

### 建造成本

建议初始成本：

- 木材：120
- 石材：60
- 金币：150

说明：

- 成本不宜过低，否则贸易站会成为早期开局最优解
- 成本也不宜过高，否则中期输出口仍然打不开

### 升级意义

贸易站升级不增加新资源种类，只强化以下能力：

- 每轮最大可成交量
- 交易效率
- 保留库存调节空间

V1 不做复杂商人事件、市场价格波动、稀有货物交易。

## Trading Model

### 资源保留线

每类资源有独立保留线：

- 木材保留
- 石材保留
- 食物保留

默认建议值：

- 木材：120
- 石材：80
- 食物：100

这些数值应支持玩家手动调整。

### 交易偏好

V1 保留 4 种贸易偏好：

1. 均衡贸易
   - 三类资源按盈余比例平均出售
2. 木材优先
   - 优先出售木材盈余
3. 石材优先
   - 优先出售石材盈余
4. 食物保守
   - 食物只在高盈余时出售，避免生存风险

### 与全局策略预设的关系

贸易系统不覆盖玩家手动设置，但全局策略可以给出推荐偏向：

| 全局策略 | 贸易建议 |
|----------|----------|
| 生存稳态 | 食物保守，木材/石材适量出售 |
| 扩张建造 | 尽量保留木材与石材，少量出售食物盈余 |
| 科研冲刺 | 出售更多木材/石材盈余，转金币支持刷新和装备 |
| 备战 Boss | 保留食物与石材，出售木材盈余 |

V1 中建议做成：

- 切换全局策略时，只给出“推荐贸易偏好”
- 不强行覆盖玩家已设置的贸易参数

这样可以保持规划感，而不是变成黑箱自动化。

## Economy Loop

### 新循环

```text
伐木场 / 采石场 / 农场 / 采集
  ->
基础资源进入库存
  ->
贸易站检测保留线以上的盈余资源
  ->
自动转为金币
  ->
金币投入招募与装备
  ->
角色数量 / 战斗力提升
  ->
更高资源产出与更高阶段 Boss 准备
```

### 新的资源分层

V1 经济系统收束为三层：

1. 基础资源
   - 木材
   - 石材
   - 食物

2. 通用货币
   - 金币

3. 阶段突破资源
   - 核心零件

这三层分别承担：

- 基础资源：生产与建造
- 金币：招募与装备
- 核心零件：Boss 突破与关键升级

## Formulas

### 可交易盈余

```text
tradable_amount = max(0, current_stock - reserve_stock - pending_buffer)
```

| Variable | Type | Range | Source | Description |
|----------|------|-------|--------|-------------|
| current_stock | int | 0+ | 资源系统 | 当前库存 |
| reserve_stock | int | 0+ | 玩家设置 | 最低保留库存 |
| pending_buffer | int | 0+ | 建造/升级队列 | 为已排队建设预留的缓冲量 |

### 单轮交易金币

```text
gold_gain_per_cycle =
sum(traded_wood * wood_price +
    traded_stone * stone_price +
    traded_food * food_price)
* efficiency_multiplier
```

建议基础价格：

- 木材：1 木材 = 1 金币
- 石材：1 石材 = 1 金币
- 食物：2 食物 = 1 金币

建议初始效率：

- Lv.1 贸易站：100%
- Lv.2 贸易站：115%
- Lv.3 贸易站：130%

### 每轮最大成交量

```text
max_trade_per_cycle = base_trade_limit + level_bonus
```

建议值：

- Lv.1：每轮最多成交 20 单位资源
- Lv.2：每轮最多成交 35 单位资源
- Lv.3：每轮最多成交 50 单位资源

这样可以避免前期直接把大量资源瞬间换成金币。

## UI Requirements

### 贸易站面板

V1 面板应至少包含：

- 当前等级
- 当前贸易偏好
- 当前三类资源库存
- 三类资源保留线
- 下轮预计金币收入
- 当前每分钟金币产出
- 自动交易开关

### 反馈重点

地图与面板需要明确表达：

- 贸易站是否已启用
- 正在交易哪类资源
- 本轮交易获得了多少金币

建议浮字：

- `贸易站 +12 金币`
- `卖出木材 x8`
- `卖出石材 x5`

### 新手引导提示

当玩家第一次资源明显溢出时，可给出提示：

`资源开始堆积。建造贸易站可以把盈余资源转成金币，用于招募和装备。`

## Edge Cases

| Scenario | Expected Behavior | Rationale |
|----------|------------------|-----------|
| 食物库存接近保留线 | 停止出售食物 | 防止贸易系统破坏生存稳态 |
| 玩家排了高成本建筑升级 | 交易时预留排队缓冲，不卖掉关键建材 | 避免“刚攒够又被系统卖掉” |
| 没有招募站和装备入口 | 仍可积累金币，但提示当前金币用途有限 | 保持系统自洽 |
| 金币严重溢出 | 不自动转成别的资源 | V1 保持经济方向单向简洁 |
| 玩家关闭自动交易 | 贸易站暂停工作 | 保留手动规划权 |

## Out of Scope

以下内容不进入 V1 贸易系统：

- 地板
- 墙体
- 道路
- 围栏
- 市场价格实时波动
- 商人事件
- 多货币体系
- 奢侈品与高阶商品加工链

这些内容统一作为 V1.2 以后扩展方向。

## Dependencies

| System | Direction | Nature of Dependency |
|--------|-----------|---------------------|
| 资源系统 | This depends on 资源系统 | 读取当前库存并计算盈余 |
| 建筑系统 | This depends on 建筑系统 | 贸易站是建筑入口 |
| 招募系统 | 招募 depends on this | 金币提供稳定招募来源 |
| 装备系统 | 装备 depends on this | 金币承担装备获取成本 |
| 全局策略系统 | This relates to 全局策略 | 提供贸易偏好建议但不强制覆盖 |

## Tuning Knobs

| Parameter | Current Value | Safe Range | Effect of Increase | Effect of Decrease |
|-----------|--------------|------------|-------------------|-------------------|
| 木材价格 | 1 | 1-2 | 更鼓励木材导向生产 | 木材更偏建造资源 |
| 石材价格 | 1 | 1-2 | 更鼓励石材导向生产 | 石材更偏建造资源 |
| 食物价格 | 0.5 | 0.5-1 | 更鼓励农业与食物贸易 | 降低卖粮冲动 |
| 每轮成交上限 | 20 | 15-40 | 加快金币流速 | 保持金币稀缺 |
| 交易效率 | 1.0 | 0.8-1.4 | 强化贸易站价值 | 让金币更难获取 |
| 默认保留库存 | 木120/石80/食100 | 视节奏调整 | 提高稳定性 | 提高激进经济玩法 |

## Acceptance Criteria

- [ ] V1 拥有唯一正式交易入口：贸易站
- [ ] 贸易系统只处理木材、石材、食物三类基础资源
- [ ] 自动交易不会卖空资源
- [ ] 玩家可设置最低保留库存
- [ ] 金币成为招募与装备的稳定来源
- [ ] 贸易系统不会抢走建造与 Boss 的主循环地位
- [ ] 地板、墙、道路、围栏明确后置到 V1 之后
