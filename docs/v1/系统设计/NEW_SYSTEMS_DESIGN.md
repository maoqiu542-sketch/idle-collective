# Idle Collective V1 聚落系统设计文档

> **Status**: Draft  
> **Author**: Codex  
> **Last Updated**: 2026-03-28  
> **Implements Pillar**: 低操作负担下的可规划聚落成长

## Overview

本文件定义 V1 的聚落核心系统，包括资源、建筑、招募站、科技、聚落宜居度与聚落发展度。V1 不追求大而全，而是围绕“玩家做规划、角色自动执行”的主轴，建立一个足够聚焦的成长结构。

## Player Fantasy

玩家应感受到：

- “我的聚落正在自动运转，但它不是无脑运转。”
- “我修的建筑不只是解锁页面，它会真实改变聚落状态。”
- “宜居度和发展度是两个不同方向的成长杠杆。”

## Detailed Design

### Core Resources

V1 只保留五类资源：

| 资源 | 作用 |
|------|------|
| 木材 | 基础建造与部分升级 |
| 石材 | 建造、升级、防御相关建筑 |
| 食物 | 生存维持 |
| 金币 | 招募与交易 |
| 核心零件 | Boss 掉落的关键成长材料 |

### Core Buildings

| 建筑 | 主功能 | 宜居度贡献 |
|------|--------|-----------|
| 房屋 | 提供人口上限与休息恢复 | +12 |
| 厨房 | 提升食物利用率与恢复效率 | +6 |
| 农场 | 稳定生产食物 | +4 |
| 伐木机 | 自动生产木材 | +2 |
| 采石机 | 自动生产石材 | +2 |
| 研究台 | 产生研究进度 | +3 |
| 招募站 | 常驻查看商队带来的候选成员 | +4 |
| 仓库 | 提高资源存储效率 | +2 |
| 兵营 | 战斗准备与训练 | +0 |

说明：

- 建筑既有直接功能，也有宜居度贡献
- 宜居度不会替代建筑主功能，而是额外提供聚落层修正

### 招募站

招募站是 V1 唯一正式招募入口。

规则：

1. 招募站常驻可查看
2. 候选成员由商队周期刷新
3. 每次刷新出现 2-4 名成员
4. 成员职业、价格、质量受：
   - 聚落宜居度
   - 聚落发展度
   - 当前候选池规则
5. 玩家可支付金币提前请求下一轮刷新

### 科技分支

V1 只保留三条科技分支：

| 分支 | 作用 |
|------|------|
| 生存科技 | 强化食物、休息、宜居与基础恢复 |
| 生产科技 | 强化采集、建造、仓储与研究效率 |
| 战备科技 | 强化训练、战斗准备、Boss 对抗能力 |

科技的职责不是加一堆小数值，而是：

- 解锁关键建筑升级
- 改变聚落规划方向
- 作为发展度组成部分

### 聚落宜居度

聚落宜居度是 V1 的统一民生数值。

影响：

- 需求恶化速度
- 工作效率
- 招募候选质量

不影响：

- Boss 前准备恢复
- Boss 难度

### 聚落发展度

聚落发展度是 V1 的阶段控制数值。

影响：

- Boss 难度池
- 招募质量上限
- 科技层级与成长节奏判断

### Building Upgrade Role

建筑升级消耗：

- 基础资源
- 核心零件

升级意义：

- 强化主功能
- 推动发展度
- 打开下一阶段规划选项

## Formulas

### 招募质量评分

```text
recruit_quality_score = livability * 0.4 + development * 0.6
```

| Variable | Type | Range | Source | Description |
|----------|------|-------|--------|-------------|
| livability | int | 0-100 | 聚落宜居度 | 反映聚落吸引力 |
| development | int | 0-100 | 聚落发展度 | 反映聚落成长阶段 |

### 招募站刷新人数

```text
candidates = 2 + floor(random_0_1 * 3)
```

| Variable | Type | Range | Source | Description |
|----------|------|-------|--------|-------------|
| random_0_1 | float | 0-0.999 | 随机值 | 用于决定 2-4 人刷新 |

### 研究进度

```text
research_progress_per_min =
base_output *
assigned_worker_count *
(1 + production_bonus + preset_bonus)
```

| Variable | Type | Range | Source | Description |
|----------|------|-------|--------|-------------|
| base_output | float | 1-5 | 建筑配置 | 研究台基础产出 |
| assigned_worker_count | int | 0-3 | AI/建筑 | 当前研究岗位人数 |
| production_bonus | float | 0-1 | 科技加成 | 生产科技带来的提升 |
| preset_bonus | float | 0-0.5 | 全局策略 | 科研冲刺时的额外修正 |

### 建筑升级成本

```text
upgrade_cost = base_cost * level_multiplier + core_part_cost
```

| Variable | Type | Range | Source | Description |
|----------|------|-------|--------|-------------|
| base_cost | int | 20-300 | 建筑配置 | 升级基础资源成本 |
| level_multiplier | float | 1.0-2.5 | 等级曲线 | 建筑等级带来的倍率 |
| core_part_cost | int | 0-8 | 建筑配置 | 升级所需核心零件 |

## Edge Cases

| Scenario | Expected Behavior | Rationale |
|----------|------------------|-----------|
| 没有房屋却继续招募 | 禁止招募，提示人口上限不足 | 维持聚落结构合理性 |
| 宜居度很低但发展度很高 | 候选成员质量受发展度托底，但不会过高 | 保证两个系统都有效 |
| 招募站存在但没有金币 | 可查看候选，但无法购买 | 提供目标感 |
| 玩家只堆生产不做研究 | 发展度提升有限，Boss 难度与成长节奏停滞 | 强制三条线协同 |
| 玩家只打 Boss 不做建设 | 核心零件累积但缺乏升级入口 | 保证材料必须进入成长 |

## Dependencies

| System | Direction | Nature of Dependency |
|--------|-----------|---------------------|
| 核心设计 | This depends on 核心设计 | 定义 V1 范围边界 |
| AI系统 | This depends on AI | 负责把岗位与策略落地 |
| Boss系统 | Boss depends on this | 发展度和核心零件与 Boss 相连 |
| 战斗属性系统 | This depends on 战斗属性 | 兵营与战备依赖战斗属性 |

## Tuning Knobs

| Parameter | Current Value | Safe Range | Effect of Increase | Effect of Decrease |
|-----------|--------------|------------|-------------------|-------------------|
| 房屋宜居度贡献 | 12 | 8-16 | 强化建设反馈 | 降低房屋战略价值 |
| 招募刷新人数下限 | 2 | 1-3 | 提高选择空间 | 更稀缺 |
| 招募刷新人数上限 | 4 | 3-5 | 提高随机波动 | 更稳定 |
| 研究台基础产出 | 1 | 1-3 | 加快科技推进 | 拉长中期节奏 |
| 建筑升级零件成本 | 2-8 | 1-10 | 强化 Boss 门槛感 | 降低 Boss 重要性 |

## Acceptance Criteria

- [ ] V1 核心资源仅有 5 类
- [ ] V1 核心建筑仅有 9 类
- [ ] 招募站是唯一正式招募入口
- [ ] 建筑同时具有主功能和宜居度贡献
- [ ] 宜居度仅影响需求恶化、工作效率、招募质量
- [ ] 发展度由研究、建筑、核心零件和战力共同组成
- [ ] 科技分支压缩为 3 条且都有明确职责
