**创建日期**：2026-03-30

**适用项目**：idle-collective

## 📋 文档说明

本文档为 `Idle Collective` 的**贸易站与金币经济系统策划案**，面向产品、策划、程序、UI、美术、测试团队，描述贸易站系统与金币经济在 V1 中的功能定位、规则设计、用户体验和验收边界。

**本文档聚焦功能策划与数值方向，不涉及具体技术实现细节。**

---

## 一、系统概述

### 1.1 系统定位

贸易站与金币经济系统是游戏中期的**资源输出与经济转化系统**。  
其核心作用不是增加新的主玩法，而是将中期溢出的基础资源转化为有规划价值的金币，再通过金币推动招募与装备成长，补足主循环中的经济出口。

该系统是连接以下系统的重要中枢：

- 资源系统
- 建筑系统
- 招募系统
- 装备系统
- Boss 阶段推进系统

### 1.2 核心价值

| 价值维度 | 描述 |
|---------|------|
| **玩家价值** | 让中期资源不再堆积无用，提供“保库存还是卖盈余”的规划决策 |
| **系统价值** | 把资源生产结果重新接回招募与装备成长，增强现有主循环 |
| **节奏价值** | 缓解中期资源出口不足，避免主循环在建造后期出现停滞 |

### 1.3 设计目标

| 目标 | 具体要求 |
|-----|---------|
| **低操作负担** | 以自动交易为主，玩家只需设置少量规则，不做高频点击卖货 |
| **经济清晰** | 玩家能明确看懂“卖了什么、赚了多少、钱花到了哪里” |
| **不抢主循环** | 贸易系统只做经济输出口，不替代建造、科技、Boss 的核心地位 |
| **可控扩展** | V1 只做贸易站，不扩展墙、地板、道路、市场事件等次级内容 |

### 1.4 系统关联

**上游系统（向贸易系统提供资源）**：
- 采集系统 → 提供木材、石材、食物
- 生产建筑系统 → 提供稳定资源盈余
- 建筑升级系统 → 影响资源产能与库存压力

**下游系统（消耗金币）**：
- 招募系统 → 购买候选成员
- 招募刷新系统 → 支付金币刷新
- 装备系统 → 抽取或购买装备

**平行系统（共同影响经济节奏）**：
- 聚落宜居度系统 → 影响招募质量
- 聚落发展度系统 → 影响阶段节奏
- Boss 系统 → 核心零件仍作为阶段突破资源

## 二、功能需求

### 2.1 核心功能模块

#### 模块一：贸易站建筑

**功能描述**：  
贸易站是 V1 唯一正式交易入口，用于自动将盈余基础资源转换为金币。

**建筑定位**：

| 项目 | 说明 |
|-----|------|
| 建筑名称 | 贸易站 |
| 建筑类型 | 后端经济建筑 |
| 是否直接产资源 | 否 |
| 是否需要值班 | 否 |
| 主要作用 | 将盈余木材、石材、食物转为金币 |
| 宜居度贡献 | +2 |

**解锁前置**：
- 仓库已建成
- 生产科技 1 级已解锁

**建造成本**：

| 资源 | 数值 |
|-----|------|
| 木材 | 120 |
| 石材 | 60 |
| 金币 | 150 |

---

#### 模块二：自动交易

**功能描述**：  
贸易站建成后自动运行，系统根据当前库存、保留库存和排队缓冲，自动出售盈余资源。

**可交易资源**：
- 木材
- 石材
- 食物

**不可交易资源**：
- 核心零件
- 装备
- 角色

**交易规则**：

```text
可交易数量 = 当前库存 - 保留库存 - 排队缓冲
```

当可交易数量小于等于 0 时，不进行出售。

---

#### 模块三：贸易偏好

**功能描述**：  
玩家可设置贸易站的资源出售倾向。

**V1 支持的偏好模式**：

| 偏好模式 | 说明 |
|---------|------|
| 均衡贸易 | 三类资源按盈余比例平均出售 |
| 木材优先 | 优先出售木材盈余 |
| 石材优先 | 优先出售石材盈余 |
| 食物保守 | 仅在食物高盈余时出售，优先保证生存 |

**设计原则**：
- 全局策略可以给出推荐，但不强制覆盖玩家设置
- 偏好模式不改变建筑与 Boss 主循环，仅影响经济流向

---

#### 模块四：保留库存

**功能描述**：  
为防止系统卖空资源，玩家可为每类资源设定最低保留库存。

**默认保留库存**：

| 资源 | 默认保留值 |
|-----|-----------|
| 木材 | 120 |
| 石材 | 80 |
| 食物 | 100 |

**设计作用**：
- 保护基础建造节奏
- 保护生存稳态
- 防止“刚攒够资源又被卖掉”

---

#### 模块五：金币用途

**功能描述**：  
金币是 V1 的中期规划货币，用于把资源积累转化为成长选择。

**主要消耗出口**：

| 用途 | 定位 |
|-----|------|
| 购买候选成员 | 核心金币 sink |
| 手动刷新招募站 | 主动金币 sink |
| 抽取或购买装备 | 中后期金币 sink |

**明确不承担的职责**：
- 不作为建筑升级主资源
- 不代替核心零件的阶段突破地位

---

### 2.2 功能优先级

| 优先级 | 功能模块 | 功能点 | 建议开发周期 |
|--------|---------|--------|-------------|
| P0 | 贸易站建筑 | 建筑解锁、建造、开关状态 | Sprint Trade-1 |
| P0 | 自动交易 | 盈余判断、成交规则、金币产出 | Sprint Trade-1 |
| P0 | 金币出口 | 招募购买、刷新消耗接通 | Sprint Trade-1 |
| P1 | 贸易偏好 | 4 种偏好模式 | Sprint Trade-2 |
| P1 | 保留库存 | 三类资源保留线调节 | Sprint Trade-2 |
| P1 | 交易反馈 | 面板、浮字、引导提示 | Sprint Trade-2 |
| P2 | 贸易站升级 | 成交上限与效率成长 | Sprint Trade-3 |

## 三、操作流程

### 3.1 贸易站解锁与建造流程

```text
玩家建成仓库
    ↓
解锁生产科技 1
    ↓
建造列表出现贸易站
    ↓
玩家支付木材、石材、金币
    ↓
贸易站进入施工
    ↓
完工后自动启用
```

**用户体验要点**：
- 贸易站应作为“中期经济工具”出现，而不是新手一上来就能造
- 解锁时需要有明确提示：“资源开始堆积，贸易站可将盈余资源转为金币”

---

### 3.2 自动交易流程

```text
贸易站已启用
    ↓
系统检测三类资源当前库存
    ↓
扣除保留库存与排队缓冲
    ↓
按当前偏好选择出售顺序
    ↓
换算为金币
    ↓
显示交易反馈
```

**用户体验要点**：
- 玩家必须能知道系统卖了什么
- 玩家必须能知道为什么某类资源没有卖
- 不能让玩家产生“系统偷了我资源”的误解

---

### 3.3 金币消耗流程

```text
贸易站产出金币
    ↓
玩家打开招募站 / 装备入口
    ↓
选择购买成员 / 刷新 / 购买装备
    ↓
金币扣除
    ↓
获得新成员或装备
    ↓
反向提升生产与战备能力
```

**用户体验要点**：
- 金币来源与去向要形成闭环感
- 让玩家感受到“卖资源是为了更强的聚落规划”，不是单纯攒钱

## 四、界面与反馈设计

### 4.1 贸易站面板信息

贸易站面板至少应展示以下内容：

| 区域 | 内容 |
|-----|------|
| 顶部 | 当前等级、启用状态 |
| 中部 | 当前偏好、三类资源库存 |
| 中部 | 三类资源保留库存 |
| 中部 | 当前每分钟金币产出 |
| 底部 | 下轮预计收益、开关按钮 |

### 4.2 地图反馈

地图或浮字需要展示：
- `贸易站 +12 金币`
- `卖出木材 x8`
- `卖出石材 x5`

### 4.3 引导反馈

推荐提示文案：

`资源开始堆积。建造贸易站可以把盈余资源转成金币，用于招募和装备。`

## 五、数值方案

### 5.1 资源价格

| 资源 | 建议价格 |
|-----|---------|
| 木材 | 1.0 - 1.2 金币 |
| 石材 | 1.0 - 1.2 金币 |
| 食物 | 0.4 - 0.6 金币 |

**设计说明**：
- 木材和石材作为常见建材，不宜过高
- 食物必须天然更不适合出售，以保护生存

### 5.2 成交节奏

| 参数 | 建议范围 |
|------|---------|
| 每轮交易周期 | 15 - 30 秒 |
| Lv.1 每轮成交上限 | 20 |
| Lv.2 每轮成交上限 | 35 |
| Lv.3 每轮成交上限 | 50 |

### 5.3 贸易站升级收益

| 等级 | 交易效率 |
|------|---------|
| Lv.1 | 1.00 |
| Lv.2 | 1.15 |
| Lv.3 | 1.30 |

## 六、异常处理与限制说明

### 6.1 常见异常场景

| 场景 | 用户提示 | 处理方式 |
|-----|---------|---------|
| 食物接近保留线 | “食物库存接近安全线，已停止出售食物。” | 立即停止卖粮 |
| 队列中有高成本建造 | “已为建造队列预留资源，本轮不出售关键材料。” | 扣除排队缓冲后再计算 |
| 金币用途不足 | “当前金币用途有限，建议优先用于招募或刷新。” | 引导至招募站 |
| 未建成仓库或未解锁科技 | “需先建成仓库并完成生产科技 1。” | 禁止建造贸易站 |

### 6.2 范围限制

V1 明确不包含以下内容：

- 墙体系统
- 地板系统
- 道路系统
- 围栏系统
- 市场价格波动
- 商人事件
- 多货币体系
- 奢侈品加工链
- 手动频繁卖货玩法

## 七、验收标准

### 7.1 功能验收

- [ ] 贸易站解锁条件正确
- [ ] 贸易站能正常建造与启用
- [ ] 自动交易逻辑正确
- [ ] 不会卖空资源
- [ ] 金币可正常流入招募与装备
- [ ] 存档读档后贸易状态一致

### 7.2 体验验收

- [ ] 玩家能理解贸易站的用途
- [ ] 玩家能看懂本轮卖出与收益
- [ ] 玩家不会误以为系统偷卖资源
- [ ] 玩家能感知金币对招募和装备的推动作用

### 7.3 平衡验收

- [ ] 贸易站不是最优开局建筑
- [ ] 贸易系统不会压过建造、科技、Boss
- [ ] 食物交易不会破坏生存稳态
- [ ] 金币不会在中期快速失控

---

## 八、结论

贸易站与金币经济系统在 V1 中应被视为**中期经济输出口系统**。  
它的职责是把资源溢出重新转化为招募与装备成长动力，而不是替代现有主循环。

若要进入 V1，必须以 **Trade MVP** 的形式落地，并严格控制边界。
