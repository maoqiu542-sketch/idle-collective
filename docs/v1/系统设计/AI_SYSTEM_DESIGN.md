# Idle Collective V1 AI系统设计文档

> **Status**: Draft  
> **Author**: Codex  
> **Last Updated**: 2026-03-28  
> **Implements Pillar**: 玩家做规划，角色自动执行

## Overview

AI 系统是 `Idle Collective V1` 的核心实现层。玩家不直接分配每个角色去哪个岗位，而是通过角色级技能优先级和全局策略预设规定聚落方向，AI 根据需求、任务权重、角色天赋与阶段目标自动分配具体行为。

## Player Fantasy

玩家应感受到：

- “我是在制定规则，而不是盯着人干活。”
- “角色会按照我的规划自己运转。”
- “当我切换策略后，聚落整体行为会明显变化。”

## Detailed Design

### Core Rules

1. AI 的调度输入来自四部分：
   - 角色状态
   - 角色技能优先级
   - 当前全局策略预设
   - 当前环境需求与任务池
2. 角色技能优先级是长期规则：
   - 与角色天赋和成长挂钩
   - 用于塑造角色个体分工倾向
3. 全局策略预设是短中期规则：
   - 一次性覆盖所有角色的技能优先级权重
   - 用于表达聚落当前目标
4. 当全局策略预设激活时：
   - 玩家不能手动修改角色级优先级
   - 角色个体差异仍由天赋和职业加成体现
5. AI 的决策顺序：
   - 处理生存类紧急需求
   - 处理当前全局目标下的高优先级任务
   - 处理低优先级维持任务
6. 生存类任务永远具备最高打断权：
   - 进食
   - 休息
   - 恢复
7. 玩家理想的操作频率是按阶段调整策略，而不是持续点工。

### Strategy Presets

#### 生存稳态

- 提高进食、休息、基础采集、烹饪权重
- 降低研究、训练与非必要建造权重

#### 扩张建造

- 提高建造、采石、伐木权重
- 保持基础生存任务不掉线
- 暂时压低研究和训练

#### 科研冲刺

- 提高研究、基础供给权重
- 降低非关键扩建
- 战备维持最低必要水平

#### 备战 Boss

- 提高训练、恢复、装备准备权重
- 保持食物和休息稳定
- 压低非关键研究与扩建

### Supported Skill Priorities

V1 保留技能级优先级：

- 采集
- 建造
- 种植
- 烹饪
- 研究
- 战斗

### States and Transitions

| State | Entry Condition | Exit Condition | Behavior |
|-------|----------------|----------------|----------|
| 待机 | 无可执行任务 | 获得有效任务 | 等待调度 |
| 移动 | 被分配目标任务 | 抵达目标点 | 前往任务点 |
| 工作 | 到达目标并开始任务 | 任务完成/被打断 | 执行采集、建造、研究等 |
| 进食 | 饥饿进入临界 | 饥饿恢复到安全值 | 优先消耗食物 |
| 休息 | 精力进入临界 | 精力恢复到安全值 | 进入房屋或恢复点 |
| 恢复 | 战斗后或受伤后 | 恢复结束 | 优先恢复战斗状态 |

### Task Arbitration

AI 在同一时间只给角色分配一个主任务，任务评分由以下因素组成：

- 基础任务权重
- 当前策略预设修正
- 角色技能优先级修正
- 角色职业/天赋修正
- 需求紧急修正
- 距离修正

### Interactions with Other Systems

| System | Input to AI | Output from AI | Responsibility |
|--------|-------------|----------------|----------------|
| 需求系统 | 当前需求值、临界状态 | 生存任务选择 | 需求提供压力，AI 处理响应 |
| 建筑系统 | 可用建筑岗位 | 岗位占用与工作行为 | 建筑提供工作位，AI 选择谁去 |
| 科技系统 | 研究目标、研究位 | 研究行为 | AI 决定何时投入研究 |
| Boss系统 | 备战窗口、挑战条件 | 训练与恢复行为 | AI 在备战策略下调整战斗准备 |

## Formulas

### 任务评分

```text
task_score =
base_weight *
preset_weight *
character_priority_weight *
profession_weight *
need_weight *
distance_weight
```

| Variable | Type | Range | Source | Description |
|----------|------|-------|--------|-------------|
| base_weight | float | 0.5-3.0 | 任务配置 | 任务原始权重 |
| preset_weight | float | 0.5-2.0 | 全局策略 | 当前策略对该任务的修正 |
| character_priority_weight | float | 0.6-1.8 | 角色优先级 | 角色对该技能的长期偏好 |
| profession_weight | float | 0.8-1.5 | 职业与天赋 | 职业匹配与天赋修正 |
| need_weight | float | 1.0-3.0 | 需求系统 | 生存类需求可拉高权重 |
| distance_weight | float | 0.6-1.0 | 距离计算 | 任务越远越低 |

### 角色优先级权重换算

```text
character_priority_weight = 2.0 - priority * 0.14
```

| Variable | Type | Range | Source | Description |
|----------|------|-------|--------|-------------|
| priority | int | 1-10 | 玩家设置 | 1 为最高优先级 |

### 预设覆盖规则

```text
effective_priority = preset_priority
```

当全局策略预设激活时，角色级手动优先级不参与运算。

## Edge Cases

| Scenario | Expected Behavior | Rationale |
|----------|------------------|-----------|
| 玩家启用全局策略预设 | 锁定角色级优先级编辑 | 避免规则打架 |
| 玩家关闭全局策略预设 | 恢复角色级优先级编辑 | 保留长期个性配置 |
| 角色需求临界 | 直接插队处理生存任务 | 挂机游戏不能让 AI 蠢死 |
| 当前策略没有可执行任务 | AI 回退到维持型任务 | 保持聚落持续运转 |
| 两个任务评分相同 | 优先距离更近者，再按创建时间排序 | 保证结果稳定 |
| 所有角色都不适合当前关键任务 | 系统提示玩家当前策略与队伍配置不匹配 | 把问题转化为规划反馈 |

## Dependencies

| System | Direction | Nature of Dependency |
|--------|-----------|---------------------|
| 核心设计 | This depends on 核心设计 | 定义 AI 服务的玩家体验 |
| 建筑系统 | This depends on 建筑 | 建筑提供岗位和功能目标 |
| 宜居度系统 | This depends on 宜居度 | 影响需求与效率 |
| Boss系统 | Boss depends on AI | 备战和恢复行为由 AI 落地 |

## Tuning Knobs

| Parameter | Current Value | Safe Range | Effect of Increase | Effect of Decrease |
|-----------|--------------|------------|-------------------|-------------------|
| 生存任务基准权重 | 2.2 | 1.8-3.0 | 更稳，不易崩盘 | 更容易因发展任务失控 |
| 预设修正最大值 | 2.0 | 1.5-2.5 | 策略切换反馈更强 | 策略切换反馈变弱 |
| 距离修正下限 | 0.6 | 0.4-0.8 | 更偏本地任务 | 更愿意跨图处理 |
| 需求打断阈值 | 临界值 | 15-35 | 更早处理需求 | 更冒险 |

## Acceptance Criteria

- [ ] 玩家能在不逐人排班的情况下驱动聚落运转
- [ ] 启用全局策略预设时，角色级优先级编辑被锁定
- [ ] 关闭全局策略预设时，角色级优先级恢复可用
- [ ] 切换四套策略后，3-10 分钟内聚落行为差异明显
- [ ] 生存任务在挂机状态下始终优先于非必要发展任务
- [ ] 角色职业与天赋会影响同一策略下的执行倾向
