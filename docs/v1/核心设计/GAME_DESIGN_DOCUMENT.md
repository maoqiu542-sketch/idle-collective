# Idle Collective V1 游戏策划案

> **Status**: Draft  
> **Author**: Codex  
> **Last Updated**: 2026-03-28  
> **Implements Pillar**: 规划型挂机聚落体验

## Overview

`Idle Collective V1` 是一款轻量自治聚落挂机规划游戏。玩家不逐个控制角色，而是通过阶段目标、角色优先级与全局策略预设设定聚落方向，由角色自动执行采集、建造、研究和备战工作。V1 的目标不是实现完整大规模模拟，而是在 8-12 人规模下做出清晰、可挂机、可规划、可验证乐趣的聚落循环。

## Player Fantasy

玩家应当感受到：

- “我像一个聚落规划者，而不是一个点工头。”
- “我改了规则，聚落就真的朝我想要的方向运转。”
- “我离开一段时间回来，会看到新的瓶颈与新的成长选择。”
- “Boss 是我扩张中的阶段门槛，而不是独立小游戏。”

## Detailed Design

### Core Rules

1. 玩家通过两种方式控制聚落：
   - 角色级技能优先级
   - 全局策略预设
2. 当启用全局策略预设时：
   - 角色级技能优先级被预设整套覆盖
   - 玩家不能单独修改角色优先级
3. 当关闭全局策略预设时：
   - 玩家回到角色级技能优先级模式
   - 可按角色天赋和成长单独配置
4. 角色自动处理以下工作：
   - 生存：进食、休息、恢复
   - 基础生产：采集、种植、烹饪
   - 发展：建造、研究
   - 战备：训练、Boss 准备
5. V1 的核心资源只有五类：
   - 木材
   - 石材
   - 食物
   - 金币
   - 核心零件
6. V1 的核心建筑只有九类：
   - 房屋
   - 厨房
   - 农场
   - 伐木机
   - 采石机
   - 研究台
   - 招募站
   - 仓库
   - 兵营
7. V1 只保留五个职业：
   - 采集者
   - 建造者
   - 农夫
   - 研究员
   - 战士
8. V1 的长期成长由三条线驱动：
   - 科技推进
   - 建筑升级推进
   - Boss 掉落的核心零件推进
9. 装备仅服务战斗准备，不作为主循环核心。

### V1 Core Loop

```text
需求压力出现
  ↓
玩家设定阶段目标与优先级
  ↓
角色自动执行生存 / 生产 / 发展 / 战备任务
  ↓
资源积累与关键建筑建设
  ↓
研究推进与发展度提升
  ↓
定时刷新更高难度 Boss
  ↓
击败 Boss 获得核心零件
  ↓
建筑升级 / 科技突破 / 进入下一轮规划
```

### New Player Experience

#### 0-10 分钟：理解核心

- 看到角色自动工作
- 理解资源变化与需求压力
- 体验第一次建筑建设
- 理解“规划比操作更重要”

#### 10-45 分钟：第一次阶段突破

- 建立研究台与招募站
- 体验第一次切换策略预设
- 理解宜居度对聚落表现的影响
- 完成第一次 Boss 准备与挑战

#### 2-4 小时：完成 V1 主成长线

- 基本聚落成型
- 科技三分支完成基础层
- 多次击败定时 Boss
- 体验 8-12 人规模下的完整循环

### Core Systems

#### 1. 角色与职业

职业决定：

- 初始技能倾向
- 调度适配效率
- 在同一预设下的角色响应差异

#### 2. 全局策略预设

V1 固定四套全局策略：

- 生存稳态
- 扩张建造
- 科研冲刺
- 备战 Boss

其本质是一组预设好的角色技能优先级规则。

#### 3. 聚落宜居度

聚落宜居度是 V1 的简化民生数值系统，主要由建筑提供。

影响：

- 角色需求恶化速度
- 角色工作效率
- 招募候选质量

#### 4. 聚落发展度

聚落发展度是 V1 的阶段控制数值。

影响：

- Boss 难度池
- 招募质量上限
- 科技推进层级

#### 5. Boss 与核心零件

Boss 定时刷新，难度随聚落发展度提升。

击败 Boss 的主要意义是获得 `核心零件`，用于：

- 建筑升级
- 科技突破
- 推动发展度继续上升

### V1 Content Scope

#### In Scope

- 聚落自动运作
- 角色优先级系统
- 全局策略预设
- 五类资源
- 九类核心建筑
- 三条科技分支
- 定时刷新 Boss
- 简化战斗属性
- 辅助型装备

#### Out of Scope

- 地形深度编辑
- 装饰系统
- 复杂幸福/社交/隐私系统
- 大规模人口模拟
- 多层稀有装备追求
- 非核心功能建筑泛滥

## Formulas

### 聚落宜居度

```text
livability = clamp(sum(building_livability) + tech_bonus, 0, 100)
```

| Variable | Type | Range | Source | Description |
|----------|------|-------|--------|-------------|
| building_livability | int | 0-20 per building | 建筑配置 | 每栋建筑提供的宜居度贡献 |
| tech_bonus | int | 0-20 | 科技 | 科技带来的额外宜居度 |

**Expected output range**: 0-100  
**Example**: 3 座房屋(36) + 厨房(6) + 农场(4) + 研究台(3) + 招募站(4) = 53

### 聚落发展度

```text
development = clamp(
  key_research_count * 20 +
  key_building_level_sum * 8 +
  core_part_spent * 2 +
  active_combat_power / 50,
  0,
  100
)
```

| Variable | Type | Range | Source | Description |
|----------|------|-------|--------|-------------|
| key_research_count | int | 0-3 | 科技系统 | 完成的关键研究数量 |
| key_building_level_sum | int | 0-15 | 建筑系统 | 关键建筑等级总和 |
| core_part_spent | int | 0-20 | 核心零件系统 | 已用于成长的核心零件数量 |
| active_combat_power | int | 0-1000 | 战斗系统 | 当前工作中或可出战角色战力总和 |

**Expected output range**: 0-100

### 宜居度对需求恶化速度修正

```text
need_decay_multiplier = clamp(1.2 - livability / 200, 0.7, 1.2)
```

| Variable | Type | Range | Source | Description |
|----------|------|-------|--------|-------------|
| livability | int | 0-100 | 聚落宜居度 | 当前聚落宜居度 |

**Expected output range**: 0.7-1.2

## Edge Cases

| Scenario | Expected Behavior | Rationale |
|----------|------------------|-----------|
| 玩家启用全局策略预设 | 锁定角色级手动优先级编辑 | 避免两套规则冲突 |
| 玩家关闭全局策略预设 | 恢复角色级优先级模式 | 保留高级玩家规划空间 |
| 资源短缺导致策略无法完全执行 | 聚落自动回退到生存维持逻辑 | 防止挂机时直接崩盘 |
| 玩家长期不打 Boss | Boss 保留，成长速度下降但不终止 | 保持挂机宽容度 |
| 宜居度很低 | 需求恶化更快、效率更低、招募质量下降 | 让建筑经营形成真实约束 |

## Dependencies

| System | Direction | Nature of Dependency |
|--------|-----------|---------------------|
| AI系统 | This depends on AI | 负责把规划转换成自动执行 |
| 建筑系统 | This depends on 建筑 | 提供功能与宜居度 |
| Boss系统 | Boss depends on this | 发展度决定难度池 |
| 科技系统 | This depends on 科技 | 关键研究参与发展度计算 |
| 装备系统 | 装备 depends on this | 装备仅作为战斗辅助 |

## Tuning Knobs

| Parameter | Current Value | Safe Range | Effect of Increase | Effect of Decrease |
|-----------|--------------|------------|-------------------|-------------------|
| 人口规模上限 | 12 | 8-12 | 增强复杂度与性能压力 | 提升聚焦度 |
| Boss 刷新周期 | 30 分钟 | 20-40 分钟 | 增加阶段压力 | 提高挂机宽容度 |
| 宜居度上限 | 100 | 80-120 | 放大建筑经营收益 | 降低建筑经营存在感 |
| 发展度关键研究权重 | 20 | 15-25 | 提高科技驱动感 | 降低科技主导程度 |
| 核心零件权重 | 2 | 1-3 | 增强 Boss 门槛感 | 降低战斗存在感 |

## Acceptance Criteria

- [ ] 玩家能在 10 分钟内理解“规划与优先级驱动聚落”
- [ ] 玩家能在 45 分钟内完成一次阶段突破
- [ ] V1 核心资源不超过 5 类
- [ ] V1 核心建筑不超过 9 类
- [ ] 全局策略预设与角色优先级规则无冲突
- [ ] Boss 主要奖励体现为核心零件，而不是装备刷取
- [ ] 8-12 人规模下核心循环完整成立
