## 一、系统概述

### 1.1 系统定位

六维系统是角色成长的核心系统，通过战斗获取经验提升六维属性等级；装备系统是角色强化的重要途径，通过穿戴装备获得属性加成。两大系统共同决定角色的战斗能力。

### 1.2 核心价值

| 维度 | 价值描述 |
|-----|---------|
| 玩家价值 | 提供清晰的角色成长路径，装备收集带来成就感 |
| 系统价值 | 与战斗系统深度关联，形成"战斗→成长→更强战斗"的闭环 |
| 商业价值 | 装备获取驱动商店消费，BOSS挑战增加活跃度 |

### 1.3 设计目标

| 目标 | 具体指标 |
|-----|---------|
| 成长感 | 每次升级有明显属性提升感知 |
| 策略性 | 装备搭配有策略选择空间 |
| 平衡性 | 不同品质装备价值差异合理 |

### 1.4 系统关联

```
战斗系统 ──(经验)──▶ 六维系统 ──(属性)──▶ 战斗系统
    │                  │
    │                  ▼
    └──────────────▶ 装备系统 ◀────────── 商店系统
                         │
                         ▼
                    战斗系统(属性加成)
```

## 三、六维属性设计

### 3.1 六维属性定义

| 属性名称 | 英文ID | 定义 | 战斗影响 |
|---------|--------|------|---------|
| 攻击力 | ATK | 角色的基础伤害能力 | 直接影响伤害输出 |
| 防御力 | DEF | 角色的伤害减免能力 | 减少受到的伤害 |
| 生命值 | HP | 角色的生命上限 | 决定生存能力 |
| 暴击率 | CRIT_RATE | 造成暴击的概率 | 0-100%概率造成暴击 |
| 暴击伤害 | CRIT_DMG | 暴击时的伤害倍率 | 暴击伤害 = 基础伤害 × 倍率 |
| 攻击速度 | ATK_SPD | 攻击的频率 | 影响每秒攻击次数 |

### 3.2 等级成长机制

**升级公式**:
```
升级所需经验 = 100 × 当前等级^1.5
```

**属性成长公式**:
```
当前属性值 = 基础属性 × (成长率 ^ 当前等级)
成长率 = 1.08
```

**等级上限**: 每项属性独立等级，最高100级

### 3.3 基础属性值

| 属性 | 基础值 |
|-----|-------|
| 攻击力 | 10 |
| 防御力 | 5 |
| 生命值 | 100 |

| 攻击速度 | 1.0 |

## 五、操作流程

### 5.1 六维升级流程

```
1. 角色参与战斗
2. 战斗结束，计算六维经验
3. 经验值累加到各属性
4. 检查是否达到升级条件
5. 若满足条件，自动升级
6. 属性值提升
7. 显示升级特效和提示
```

### 5.2 装备穿戴流程

```
1. 玩家打开角色装备界面
2. 点击装备栏位
3. 显示背包中可穿戴装备列表
4. 选择要穿戴的装备
5. 检查槽位是否匹配
6. 若槽位已有装备，先卸下旧装备
7. 穿戴新装备
8. 更新角色属性
9. 显示属性变化动画
```

### 5.3 商店购买装备流程

```
1. 打开商店界面
2. 选择"装备"标签页
3. 浏览商品列表
4. 点击想要购买的装备
5. 确认购买（检查金币是否足够）
6. 扣除金币，装备进入背包
7. 显示购买成功提示
```

## 八、异常处理与用户提示

### 8.1 常见异常场景

| 场景 | 用户提示 | 处理方式 |
|-----|---------|---------|
| 金币不足购买装备 | "金币不足！缺少xxx金币" | 弹窗提示 |
| 装备类型不匹配 | "该装备无法穿戴到此槽位" | Toast提示 |
| 背包已满 | "背包已满，请先清理" | 弹窗提示 |
| 卸下装备时背包满 | "背包已满，无法卸下装备" | Toast提示 |

### 8.2 操作限制说明

- 每个槽位只能穿戴对应类型的装备
- 同一件装备不能同时穿戴在多个槽位
- 装备穿戴后绑定角色，不可交易

## 十、验收标准

### 10.1 六维系统验收

| 功能点 | 验收标准 |
|-------|---------|
| 经验获取 | 战斗结束后正确计算并累加经验 |
| 等级提升 | 经验满足条件时自动升级 |
| 属性计算 | 升级后属性值正确提升 |
| 显示更新 | 属性面板实时显示当前值 |

### 10.2 装备系统验收

| 功能点 | 验收标准 |
|-------|---------|
| 装备购买 | 金币足够时成功购买，装备进入背包 |
| 装备穿戴 | 正确穿戴到对应槽位，属性正确加成 |
| 装备卸下 | 正确卸下装备，属性正确扣除 |
| 品质显示 | 不同品质显示对应颜色和特效 |

### 10.3 属性计算验收

| 项目 | 标准 |
|-----|------|
| 最终属性 | 基础属性 + 六维加成 + 装备加成 |
| 战力计算 | 按公式正确计算战力值 |
| 实时更新 | 装备变化后属性立即更新 |

## 十二、附录

### 12.1 名词解释

| 术语 | 解释 |
|-----|------|
| 六维属性 | 攻击力、防御力、生命值、暴击率、暴击伤害、攻击速度六项属性 |
| 装备品质 | 装备的稀有程度，决定属性倍率 |
| 战力 | 综合评估角色战斗能力的数值 |

### 12.2 战力计算公式

```
战力 = 攻击力 × 2 + 防御力 × 1.5 + 生命值 × 0.1 + 暴击率 × 10 + 暴击伤害 × 5 + 攻击速度 × 50
```
