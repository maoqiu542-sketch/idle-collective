## 一、系统概述

### 1.1 系统定位

BOSS副本系统是游戏的核心挑战内容，玩家通过挑战BOSS获取丰厚奖励，包括金币、装备和六维经验。系统与六维系统、装备系统深度关联，是玩家成长的重要驱动力。

### 1.2 核心价值

| 维度 | 价值描述 |
|-----|---------|
| 玩家价值 | 提供挑战目标和成就感，获取稀有装备和资源 |
| 系统价值 | 驱动六维成长和装备收集，形成核心玩法循环 |
| 商业价值 | 增加玩家活跃度和留存率 |

### 1.3 设计目标

| 目标 | 具体指标 |
|-----|---------|
| 挑战性 | BOSS难度与玩家成长匹配，有策略空间 |
| 公平性 | 失败惩罚合理，不影响玩家继续游戏 |
| 期待感 | BOSS出现有仪式感，奖励有吸引力 |

### 1.4 系统关联

```
六维系统 ──(属性)──▶ BOSS战斗 ◀──(装备加成)── 装备系统
    │                  │
    │                  ▼
    └────────────── 战斗结算
                         │
            ┌────────────┼────────────┐
            ▼            ▼            ▼
        金币奖励      装备掉落      六维经验
```

## 三、操作流程

### 3.1 BOSS出现流程

```
1. 游戏启动计时器
2. 每1小时触发BOSS出现
3. 主界面显示BOSS入口按钮（发光闪烁）
4. 玩家点击入口按钮
5. 打开BOSS信息面板
6. 显示BOSS属性和推荐战力
```

### 3.2 战斗准备流程

```
1. 玩家打开BOSS信息面板
2. 显示当前可用角色列表
3. 玩家选择参战角色（最多3个）
4. 显示总战力和推荐战力对比
5. 玩家确认开始战斗
6. 进入战斗场景
```

### 3.3 战斗执行流程

```
1. 战斗开始
2. 按攻击速度排序行动顺序
3. 角色攻击BOSS（计算伤害、暴击）
4. BOSS攻击角色（轮流攻击）
5. 循环直到一方生命归零
6. 战斗结束，进入结算
```

### 3.4 战斗结算流程

**胜利结算**:
```
1. 显示胜利界面
2. 计算奖励（金币、装备、经验）
3. 播放奖励动画
4. 发放奖励到玩家账户
5. 参战角色获得六维经验
6. BOSS消失，重新开始计时
```

**失败结算**:
```
1. 显示失败界面
2. 参战角色进入休息状态（15分钟）
3. 显示休息倒计时
4. BOSS保留，可再次挑战
```

## 四、战斗系统设计

### 4.1 伤害计算公式

**基础伤害**:
```
伤害 = 攻击力 × (1 - 防御力 / (防御力 + 100))
```

**暴击机制**:
```
if (随机数 < 暴击率) {
  最终伤害 = 伤害 × 暴击伤害倍率
}
```

**攻击速度**:
```
攻击间隔(秒) = 1 / 攻击速度
```

### 4.2 战斗规则

| 规则 | 说明 |
|-----|------|
| 参战人数 | 最多3个角色同时参战 |
| 行动顺序 | 按攻击速度从高到低排序 |
| 目标选择 | BOSS轮流攻击参战角色 |
| 战斗时长 | 无时间限制，直到一方死亡 |
| 逃跑机制 | 不支持逃跑 |

### 4.3 战力评估

**战力计算公式**:
```
战力 = 攻击力 × 2 + 防御力 × 1.5 + 生命值 × 0.1 + 暴击率 × 10 + 暴击伤害 × 5 + 攻击速度 × 50
```

**推荐战力显示**:
| 对比结果 | 颜色提示 | 说明 |
|---------|---------|------|
| 玩家战力 ≥ 推荐战力 × 1.2 | 绿色 | 轻松挑战 |
| 推荐战力 × 0.8 ≤ 玩家战力 < 推荐战力 × 1.2 | 黄色 | 正常挑战 |
| 玩家战力 < 推荐战力 × 0.8 | 红色 | 危险挑战 |

## 八、异常处理与用户提示

### 8.1 常见异常场景

| 场景 | 用户提示 | 处理方式 |
|-----|---------|---------|
| 无可用角色参战 | "没有可用角色，请等待角色休息结束" | 弹窗提示 |
| 角色正在休息 | "该角色正在休息中，剩余时间xx:xx" | 灰色显示，不可选择 |
| 战力过低警告 | "战力过低，挑战可能失败，是否继续？" | 二次确认弹窗 |
| 战斗中断线 | "战斗中断，请重新登录" | 战斗作废，角色不进入休息 |

### 8.2 操作限制说明

- BOSS出现后玩家不点击则一直保留
- 进入战斗后不可中途退出
- 休息中的角色不可参战

## 十、验收标准

### 10.1 功能验收

| 功能点 | 验收标准 |
|-------|---------|
| BOSS触发 | 每1小时正确触发BOSS出现 |

| 项目 | 标准 |
|-----|------|

| 异常处理 | 所有异常情况有正确处理 |
