> 基于100个游戏设计原则框架

## 一、设计概述

### 1.1 核心设计理念

基于 **RimWorld 工作优先级系统** 设计，采用 **效用AI (Utility AI)** 架构。

### 1.2 设计原则应用

| 原则 | 应用 | 效果 |
|------|------|------|
| **心流通道** | 需求衰减与任务分配平衡 | 维持角色持续活动 |
| **反馈循环** | 任务完成→资源获得→技能提升 | 强化玩家观察乐趣 |
| **有意义的选择** | 职业选择影响任务偏好 | 增加策略深度 |
| **涌现玩法** | 多角色协作产生复杂行为 | 创造独特体验 |

## 四、任务调度系统

### 4.1 三层优先级体系

```
优先级 1 (紧急):  火灾、救援、治疗
优先级 2 (高):    进食、睡眠、医疗
优先级 3 (普通):  采集、建造、种植、制作
优先级 4 (低):    清洁、娱乐、社交、冥想
```

### 4.2 效用计算公式

```
效用值 = 基础分 × 需求权重 × 距离系数 × 紧急系数 × 技能系数

其中:
- 基础分: 任务类型的固定效用值
- 需求权重: 1.0 + 紧急度 × 0.5
- 距离系数: 1.0 - (距离 / 最大距离) × 0.5
- 紧急系数: 3.0 (紧急任务)
- 技能系数: 0.5 + (技能等级 / 20)
```

### 4.3 任务中断机制

```typescript
shouldInterruptCurrentTask(): boolean {
  // 1. 检查临界需求
  if (criticalNeeds.length > 0) {
    return hasHigherPriorityReliefTask()
  }
  
  // 2. 检查紧急任务
  if (urgentTasks.length > 0) {
    return hasHigherPriorityUrgentTask()
  }
  
  return false
}
```

## 五、行为执行系统

### 5.1 任务分解

```
任务 (Task)
    │
    ├── 移动行为 (Move)
    │       └── 每帧移动一格
    │
    └── 工作行为 (Work)
            └── 累积进度直到完成
```

### 5.2 支持的行为类型

| 行为 | 执行逻辑 | 完成条件 |
|------|----------|----------|
| `move` | 向目标移动 | 距离 ≤ 1 |
| `harvest` | 采集资源 | 进度 ≥ 2000ms |
| `eat` | 进食 | 立即完成 |
| `sleep` | 睡眠 | 进度 ≥ 10000ms |
| `rest` | 休息 | 进度 ≥ 5000ms |

## 六、职业系统

### 6.1 职业任务偏好

| 职业 | 偏好任务 | 效率加成 |
|------|----------|----------|
| 采集者 | 采集、搬运 | +20% |
| 建造者 | 建造、制作 | +30% |
| 农夫 | 种植、采集 | +20% |
| 战士 | 灭火、救援 | +50% |

### 6.2 涌现玩法设计

```
职业组合 → 策略倾向

采集者 + 建造者 → 快速扩张策略
农夫 + 建造者 → 稳定发展策略
战士 + 采集者 → 探索冒险策略
```

## 八、优化建议

### 8.1 短期优化

1. **增加视觉反馈**
   - 采集动画
   - 资源飘字
   - 状态图标

2. **完善任务类型**
   - 添加建造任务
   - 添加存储任务
   - 添加社交任务

### 8.2 中期优化

1. **增加角色个性**
   - 特质系统
   - 偏好系统
   - 技能成长

2. **增加随机事件**
   - 危机事件
   - 机遇事件
   - 故事事件

### 8.3 长期优化

1. **增加玩家控制**
   - 手动任务分配
   - 区域规划
   - 优先级微调

2. **增加社交系统**
   - 角色关系
   - 社交互动
   - 团队协作
