# 新系统设计文档

## 概述

本文档详细描述建筑系统、六维系统、装备系统、BOSS系统的设计方案，包括数值设定、机制规则和实现要点。

---

## 一、建筑系统

### 1.1 生产建筑类型

| 建筑名称 | 建造成本 | 建造时间 | 生产间隔 | 产出 |
|---------|---------|---------|---------|------|
| 伐木机 | 100木材 + 50石材 + 200金币 | 60秒 | 60秒 | 10木材 |
| 采石机 | 150木材 + 80石材 + 300金币 | 90秒 | 90秒 | 6石材 |
| 牧场 | 200木材 + 100石材 + 500金币 | 120秒 | 120秒 | 4皮革 |

### 1.2 生产机制

**核心规则**：
- 生产建筑无需原料投入
- 需要角色值班才能正常生产
- 无角色值班时，生产效率降低50%
- 有角色值班时，生产效率100%

**值班系统**：
- 角色可以在建筑上执行"值班"任务
- 值班期间角色状态为 WORKING
- 值班角色获得对应天赋经验（伐木机→采集，采石机→挖矿，牧场→烹饪）

### 1.3 建造流程

1. 玩家在商店界面购买建筑（消耗资源）
2. 购买后自动创建建造任务
3. AI系统分配空闲角色执行建造
4. 建造完成后建筑激活，开始生产循环

### 1.4 建筑状态

```typescript
enum BuildingState {
  PLANNED = 'planned',      // 已规划，等待建造
  CONSTRUCTING = 'constructing', // 建造中
  ACTIVE = 'active',        // 激活状态
  INACTIVE = 'inactive',    // 停用状态
}
```

## 二、六维系统

### 2.1 六维属性定义

| 属性名称 | 英文ID | 定义 | 战斗影响 |
|---------|--------|------|---------|
| 攻击力 | ATK | 角色的基础伤害能力 | 直接影响伤害输出 |
| 防御力 | DEF | 角色的伤害减免能力 | 减少受到的伤害 |
| 生命值 | HP | 角色的生命上限 | 决定生存能力 |
| 暴击率 | CRIT_RATE | 造成暴击的概率 | 0-100%概率造成双倍伤害 |
| 暴击伤害 | CRIT_DMG | 暴击时的伤害倍率 | 暴击伤害 = 基础伤害 * 暴击伤害倍率 |
| 攻击速度 | ATK_SPD | 攻击的频率 | 影响每秒攻击次数 |

### 2.2 等级成长机制

**经验获取**：
- 每次参与战斗获得经验
- 经验值 = 基础经验 * (1 + 战斗表现加成)

**升级公式**：
```
升级所需经验 = 100 * 等级^1.5
```

**等级上限**：每项属性独立等级，最高100级

**属性成长**：
```
每级属性提升 = 基础属性 * 1.08
```

**等级经验表（前10级）**：

| 等级 | 所需经验 | 累计经验 | 属性倍率 |
|-----|---------|---------|---------|
| 1 | 100 | 100 | 1.08 |
| 2 | 283 | 383 | 1.17 |
| 3 | 520 | 903 | 1.26 |
| 4 | 800 | 1703 | 1.36 |
| 5 | 1118 | 2821 | 1.47 |
| 6 | 1469 | 4290 | 1.59 |
| 7 | 1852 | 6142 | 1.71 |
| 8 | 2262 | 8404 | 1.85 |
| 9 | 2700 | 11104 | 2.00 |
| 10 | 3162 | 14266 | 2.16 |

### 2.3 基础属性值

| 属性 | 基础值 | 来源 |
|-----|-------|------|
| 攻击力 | 10 | 角色基础 |
| 防御力 | 5 | 角色基础 |
| 生命值 | 100 | 角色基础 |
| 暴击率 | 5% | 角色基础 |
| 暴击伤害 | 150% | 角色基础 |
| 攻击速度 | 1.0 | 角色基础 |

## 三、装备系统

### 3.1 装备品质

| 品质 | 颜色 | 属性倍率 | 获取途径 |
|-----|------|---------|---------|
| 普通 | 白色 | 100% | 商店购买 |
| 精良 | 绿色 | 115% | 商店购买、低级BOSS |
| 稀有 | 蓝色 | 130% | 商店刷新、中级BOSS |
| 史诗 | 紫色 | 145% | 高级BOSS |
| 传说 | 橙色 | 160% | 稀有BOSS |

### 3.2 装备槽位

| 槽位 | 英文ID | 主要属性倾向 |
|-----|--------|-------------|
| 武器 | WEAPON | 攻击力、暴击率、暴击伤害、攻击速度 |
| 头盔 | HELMET | 防御力、生命值 |
| 铠甲 | ARMOR | 防御力、生命值 |
| 鞋子 | BOOTS | 防御力、攻击速度 |
| 饰品1 | ACCESSORY_1 | 暴击率、暴击伤害 |
| 饰品2 | ACCESSORY_2 | 暴击率、暴击伤害 |

### 3.3 装备属性范围

**武器**：
- 攻击力：5-50（随品质提升）
- 暴击率：0-10%
- 暴击伤害：0-30%
- 攻击速度：0-0.3

**防具类（头盔、铠甲、鞋子）**：
- 防御力：3-30
- 生命值：10-100

**饰品类**：
- 暴击率：2-15%
- 暴击伤害：10-50%

### 3.4 商店系统

**刷新机制**：
- 每24小时自动刷新商品
- 可消耗金币立即刷新（刷新费用：100金币）

**商品规则**：
- 每次刷新展示6件装备
- 品质概率分布：
  - 普通：40%
  - 精良：30%
  - 稀有：20%
  - 史诗：8%
  - 传说：2%

**价格公式**：
```
装备价格 = 基础价格 * 品质倍率 * (1 + 等级 * 0.1)
```

| 品质 | 基础价格范围 |
|-----|-------------|
| 普通 | 50-100金币 |
| 精良 | 150-300金币 |
| 稀有 | 400-800金币 |
| 史诗 | 1000-2000金币 |
| 传说 | 3000-5000金币 |

## 四、BOSS系统

### 4.1 BOSS触发机制

**出现规则**：
- 每1小时出现一次BOSS
- 玩家不点击入口，BOSS一直保留
- 点击入口后进入战斗，战斗结束后BOSS消失

**UI提示**：
- 主界面显示BOSS入口按钮
- BOSS可用时按钮发光闪烁
- 显示BOSS名称和推荐战力

### 4.2 BOSS属性设计

| BOSS名称 | 等级 | 生命值 | 攻击力 | 防御力 | 掉落品质倾向 |
|---------|-----|-------|-------|-------|-------------|
| 哥布林首领 | 1 | 500 | 15 | 5 | 普通-精良 |
| 石像鬼 | 5 | 1000 | 30 | 15 | 精良-稀有 |
| 暗影骑士 | 10 | 2000 | 50 | 25 | 稀有-史诗 |
| 炎魔领主 | 15 | 4000 | 80 | 40 | 史诗 |
| 远古巨龙 | 20 | 8000 | 120 | 60 | 史诗-传说 |

### 4.3 战斗系统

**战斗公式**：
```
伤害 = 攻击力 * (1 - 防御力/(防御力 + 100))
```

**暴击机制**：
```
if (random() < 暴击率) {
  最终伤害 = 伤害 * 暴击伤害倍率
}
```

**攻击速度**：
```
攻击间隔 = 1 / 攻击速度（秒）
```

**战斗流程**：
1. 选择参战角色（最多选择3个角色）
2. 角色按攻击速度轮流攻击BOSS
3. BOSS按固定间隔攻击角色
4. 任一方生命值归零则战斗结束

### 4.4 战斗结算

**胜利条件**：
- BOSS生命值归零

**胜利奖励**：
- 金币奖励 = BOSS等级 * 50
- 装备掉落（按BOSS掉落表）
- 六维经验 = BOSS等级 * 10

**失败条件**：
- 所有参战角色生命值归零

**失败惩罚**：
- 参战角色进入休息状态15分钟
- 无其他惩罚（不损失金币、装备）

### 4.5 战力评估

**战力计算公式**：
```
战力 = 攻击力 * 2 + 防御力 * 1.5 + 生命值 * 0.1 + 暴击率 * 10 + 暴击伤害 * 5 + 攻击速度 * 50
```

**推荐战力显示**：
- 显示BOSS推荐战力
- 玩家当前战力对比
- 颜色提示：绿色（推荐）、黄色（挑战）、红色（危险）

## 七、数值平衡备注

### 7.1 设计原则

1. **渐进式成长**：玩家能感受到持续的进步
2. **有意义的选择**：资源分配需要权衡
3. **风险回报平衡**：高难度BOSS给予高回报
4. **避免死胡同**：失败不会导致无法继续游戏

### 7.2 调整建议

- 初期可适当降低BOSS难度，让玩家体验战斗系统
- 装备掉落率可根据实际测试调整
- 生产建筑的产出可根据经济平衡调整
- 六维成长曲线可根据玩家反馈微调
