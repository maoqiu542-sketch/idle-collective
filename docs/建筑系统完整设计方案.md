# 建筑系统完整设计方案

> 版本 1.0 | 全面设计规划 | 适用于 Idle Collective

## 一、建筑定义与分类

### 1.1 建筑定位

建筑是玩家在游戏中进行资源生产、功能解锁、角色培养的核心载体。每个建筑占据地图上的固定位置，提供特定功能，并可通过升级提升效率。

**建筑的核心价值**：
- **生产价值**：产出资源，满足玩家发展需求
- **策略价值**：不同建筑的组合影响发展路线
- **成长价值**：升级系统提供长期目标
- **视觉价值**：装饰美化游戏世界

### 1.2 建筑分类总览

| 类别 | 核心功能 | 数量 | 代表建筑 |
|------|---------|------|---------|
| **生产类** | 产出原始资源 | 5种 | 伐木场、采石场、农场、矿场、熔炉 |
| **功能类** | 提供特殊功能 | 3种 | 研究台、仓库、训练场 |
| **居住类** | 提供人口容量 | 2种 | 草屋、木屋 |
| **防御类** | 阻挡/攻击 | 1种 | 城墙 |
| **装饰类** | 美化/增益 | 3种 | 喷泉、路灯、旗帜 |

**总计：14种建筑**

### 1.3 各类建筑详细定义

#### 1.3.1 生产类建筑

**伐木场**
- 功能：从地图资源点采集木材
- 视觉风格：原木结构、木屑散落、锯子工具
- 产出：木材 1个/10秒（基础）
- 值班需求：需要1名采集者/猎人职业角色

**采石场**
- 功能：从地图资源点采集石材
- 视觉风格：石砌结构、碎石堆、镐头
- 产出：石材 1个/10秒（基础）
- 值班需求：需要1名采集者/猎人职业角色

**农场**
- 功能：持续产出食物
- 视觉风格：木质篱笆、农作物田地、水渠
- 产出：食物 1个/15秒（基础）
- 值班需求：需要1名农夫/采集者职业角色

**矿场**
- 功能：从地图深层资源点采集矿石
- 视觉风格：矿洞入口、铁轨、矿车
- 产出：矿石 1个/12秒（基础）
- 解锁条件：生产科技Lv.1
- 值班需求：需要1名工程师/猎人职业角色

**熔炉**
- 功能：将矿石冶炼为金属锭
- 视觉风格：砖砌炉体、烟囱、金属砧
- 产出：铁锭 1个/20秒（需要2矿石）
- 解锁条件：生产科技Lv.2
- 值班需求：需要1名工程师职业角色

#### 1.3.2 功能类建筑

**研究台**
- 功能：产出科技点数
- 视觉风格：书桌、书架、烛台/灯泡
- 产出：科技点 1点/分钟（基础）
- 值班需求：需要1名学者/任意职业角色

**仓库**
- 功能：增加资源存储上限
- 视觉风格：木制大型储物柜、货架
- 效果：存储上限 +500（每级）
- 特殊：无须值班，被动生效

**训练场**
- 功能：提升角色战斗经验获取速度
- 视觉风格：木制训练假人、武器架、沙袋
- 效果：值班角色战斗经验 +50%
- 解锁条件：战斗科技Lv.1
- 值班需求：需要1名战士/猎人职业角色

#### 1.3.3 居住类建筑

**草屋**
- 功能：提供人口容量
- 视觉风格：茅草屋顶、木质框架
- 效果：人口上限 +2
- 特殊：无须值班，被动生效

**木屋**
- 功能：提供更多人口容量，附加舒适度加成
- 视觉风格：木板屋顶、窗户、小花园
- 效果：人口上限 +3，角色需求衰减速度 -10%
- 解锁条件：建筑科技Lv.1
- 特殊：无须值班，被动生效

#### 1.3.4 防御类建筑

**城墙**
- 功能：阻挡敌方单位进入
- 视觉风格：石砌城墙、城门
- 效果：Boss攻击时被摧毁前保护内部建筑
- 解锁条件：建筑科技Lv.1
- 特殊：无耐久度限制，永久存在

#### 1.3.5 装饰类建筑

**喷泉**
- 功能：美化+局部资源产出加成
- 视觉风格：石制喷泉、水流动画
- 效果：周围3格内建筑产出 +5%
- 解锁条件：建筑科技Lv.3

**路灯**
- 功能：美化+夜间安全
- 视觉风格：铁艺路灯、暖黄灯光
- 效果：周围2格内角色夜间不减速
- 解锁条件：建筑科技Lv.2

**旗帜**
- 功能：宣告领地+小幅产出加成
- 视觉风格：木质旗杆、旗帜飘动
- 效果：所属领地内角色产出 +2%
- 解锁条件：初始即可建造

## 二、系统需求分析

### 2.1 技术系统依赖

每个建筑需要以下系统支持：

| 系统 | 支持内容 | 优先级 |
|------|---------|--------|
| **地图系统** | 建筑位置放置、碰撞检测、地形要求 | P0 |
| **资源系统** | 建筑消耗/产出资源、存储上限 | P0 |
| **角色系统** | 值班分配、职业匹配、效率计算 | P0 |
| **科技系统** | 建筑解锁、条件检查 | P0 |
| **升级系统** | 建筑升级、材料消耗 | P0 |
| **UI系统** | 建筑面板、生产进度、升级界面 | P0 |
| **存档系统** | 建筑状态保存/加载 | P0 |
| **音效系统** | 建造音效、生产完成音效 | P1 |
| **动画系统** | 建造动画、生产动画、产出特效 | P1 |

### 2.2 数据结构需求

```typescript
// 建筑实例数据
interface Building {
  id: string
  type: BuildingType
  position: { x: number; y: number }
  level: number
  state: BuildingState
  assignedCharacterIds: string[]
  progress: number  // 当前产出进度 (0-1)
  createdAt: number
}

// 建筑配置数据
interface BuildingConfig {
  type: BuildingType
  name: string
  description: string
  size: { width: number; height: number }
  cost: ResourceCost
  buildTime: number  // 毫秒
  production?: {
    interval: number  // 产出间隔
    output: ResourceType
    outputAmount: number
    requiredWorkerProfession?: ProfessionType
    workerCount?: number
  }
  effects?: BuildingEffect[]
  upgradeCosts: UpgradeCost[]
  requiresTech?: string
}
```

### 2.3 系统接口需求

| 接口 | 输入 | 输出 | 说明 |
|-----|------|------|------|
| `createBuilding(type, position)` | 建筑类型、位置 | 建筑实例 | 扣除资源，开始建造 |
| `upgradeBuilding(id)` | 建筑ID | 升级结果 | 扣除材料，提升等级 |
| `assignWorker(buildingId, charId)` | 建筑ID、角色ID | 分配结果 | 角色到建筑值班 |
| `removeWorker(buildingId, charId)` | 建筑ID、角色ID | 移除结果 | 角色离开建筑 |
| `getProductionStatus(buildingId)` | 建筑ID | 进度/产出 | 查询当前生产状态 |
| `checkBuildable(type)` | 建筑类型 | 是否可建 | 检查资源/科技条件 |
| `checkUpgradeable(buildingId)` | 建筑ID | 是否可升级 | 检查资源/条件 |

## 三、玩法机制设计

### 3.1 核心玩法循环

```
┌─────────────────────────────────────────────────────┐
│                    建筑玩法循环                        │
│                                                      │
│   地图探索 → 发现资源点                              │
│       ↓                                              │
│   收集资源 → 满足建造条件                            │
│       ↓                                              │
│   建造建筑 → 分配值班角色                            │
│       ↓                                              │
│   建筑产出 → 积累资源                                │
│       ↓                                              │
│   升级建筑 → 提升产出效率                            │
│       ↓                                              │
│   挑战Boss → 获取升级材料                            │
│       ↓                                              │
│   科技研究 → 解锁新建筑                              │
│       ↓                                              │
│   建造更强建筑 → 循环                                 │
│                                                      │
└─────────────────────────────────────────────────────┘
```

### 3.2 建造机制

**建造条件检查**：
1. 资源充足（金币+材料）
2. 科技已解锁（若有前置科技要求）
3. 地图位置可用（无碰撞、地形合适）
4. 人口未达上限（居住类建筑）

**建造流程**：
```
选择建筑类型 → 显示建造条件 →
确认放置位置 → 扣除资源 →
开始建造动画（X秒）→ 建造完成
```

**建造时间**：
| 建筑类型 | 建造时间 | 可否加速 |
|---------|---------|---------|
| 生产类 | 30秒 | 可用金币加速 |
| 功能类 | 45秒 | 可用金币加速 |
| 居住类 | 20秒 | 可用金币加速 |
| 防御类 | 60秒 | 可用金币加速 |
| 装饰类 | 10秒 | 不可加速 |

### 3.3 值班机制

**值班规则**：
- 1个角色同时只能值班1个建筑
- 1个建筑可容纳1-N名角色（视类型而定）
- 角色职业与建筑匹配时获得效率加成
- 值班期间角色无法进行其他活动

**职业匹配加成**：

| 建筑 | 最佳职业 | 效率加成 |
|-----|---------|---------|
| 伐木场 | 采集者、猎人 | +30% |
| 采石场 | 采集者、猎人 | +30% |
| 农场 | 农夫、采集者 | +30% |
| 矿场 | 工程师、猎人 | +30% |
| 熔炉 | 工程师 | +40% |
| 研究台 | 学者、工程师 | +30% |
| 训练场 | 战士、猎人 | +50% |

### 3.4 产出机制

**产出公式**：
```
实际产出 = 基础产出 × 建筑等级系数 × (1 + 职业加成) × (1 + 科技加成) × (1 + 装饰加成)
```

**等级系数**：
| 等级 | 系数 | 精华消耗 | 资源消耗 |
|-----|------|---------|---------|
| Lv.1 | 1.0x | - | - |
| Lv.2 | 1.2x | 10个 | 木材×20、石材×10 |
| Lv.3 | 1.5x | 30个 | 木材×50、铁锭×20 |
| Lv.4 | 2.0x | 80个 | 钢材×30 |
| Lv.5 | 3.0x | 200个 | 秘银×20 |

**产出间隔**：基础间隔 × (1 - 效率加成)，最低不低于基础间隔的30%

### 3.5 升级机制

**升级条件**：
1. 建筑等级 < 5
2. 建筑精华充足
3. 对应资源充足
4. 科技要求已满足（若有）

**升级流程**：
```
点击建筑 → 显示升级面板 →
检查材料 → 确认升级 →
扣除材料 → 播放升级动画 →
属性提升 → 升级完成
```

**升级效果**：
- 产出效率提升
- 视觉效果变化（更高级的外观）
- 解锁新功能（部分建筑）

### 3.6 系统关联

**与资源系统的关联**：
- 建筑消耗资源进行建造/升级
- 建筑产出资源进入仓库
- 仓库限制存储总量

**与角色系统的关联**：
- 角色提供劳动力（值班）
- 职业影响生产效率
- 角色属性影响值班效果

**与科技系统的关联**：
- 科技解锁新建筑类型
- 科技提升产出效率
- 科技降低运营成本

**与Boss系统的关联**：
- Boss掉落建筑精华（升级材料）
- Boss挑战需要战斗力支撑
- 击败Boss后解锁更多建筑可能

| 采石场 | 产石材 | 6个/分钟 | 基础生产 | ★★★★★ |
| 农场 | 产食物 | 4个/分钟 | 角色需求 | ★★★★☆ |
| 研究台 | 产科技点 | 1点/分钟 | 科技解锁 | ★★★★☆ |

### 4.4 建造优先级引导

**新手期（0-30分钟）建议建造顺序**：
1. 仓库 - 增加存储上限，保障资源积累
2. 伐木场×2 - 快速积累木材
3. 采石场×2 - 快速积累石材
4. 农场×1 - 维持角色食物需求

**发展期（30分钟-2小时）建议建造顺序**：
1. 研究台 - 产出科技点，解锁新科技
2. 矿场+熔炉 - 建立矿石加工线
3. 草屋/木屋 - 扩大人口，招募更多角色

**挑战期（2小时+）建议建造顺序**：
1. 训练场 - 强化角色战斗力
2. 城墙 - 防御Boss攻击
3. 喷泉/路灯 - 提升整体效率

## 五、玩家体验设计

### 5.1 策略选择空间

**发展路线A：资源优先**
- 优先建造多个伐木场/采石场
- 快速积累基础资源
- 后期转型生产类

**发展路线B：科技优先**
- 优先建造研究台
- 快速解锁科技
- 科技加成弥补资源不足

**发展路线C：战斗力优先**
- 优先建造训练场
- 挑战高级Boss
- Boss掉落加速发展

### 5.2 进度感设计

**短期目标（每10分钟）**：
- [ ] 建造2-3个生产建筑
- [ ] 积累100+木材
- [ ] 招募1-2个角色

**中期目标（每1小时）**：
- [ ] 升级1个建筑到Lv.2
- [ ] 解锁1项新科技
- [ ] 击败Lv.1 Boss

**长期目标（每4小时）**：
- [ ] 拥有完整产业链
- [ ] 挑战Lv.2-3 Boss
- [ ] 人口达到20+

### 5.3 成就感设计

**里程碑成就**：
- "初具规模" - 建造第10个建筑
- "自给自足" - 某资源持续产出满足消耗
- "建筑大师" - 拥有5种不同类型建筑
- "升级达人" - 首次将建筑升到Lv.3
- "领地扩张" - 建筑覆盖地图30%

**实时反馈**：
- 产出时显示 "+1 木材" 浮动文字
- 升级时播放庆祝特效+音效
- 建造完成时显示建筑名称+效果

### 5.4 引导机制

**新手引导流程**：
```
第1次进入 → 提示"点击建造按钮"
↓
选择伐木场 → 提示"在空地点击放置"
↓
建造完成 → 提示"招募角色并分配值班"
↓
首次产出 → 提示"资源已存入仓库"
↓
引导结束 → 显示"当前目标：建造更多建筑"
```

**界面引导**：
- 建筑按钮上显示未解锁图标（若有前置条件）
- 资源不足时按钮变灰+提示所需资源
- 可建造时按钮高亮+光晕提示

### 5.5 反馈机制

**视觉反馈**：
- 建造中：脚手架+进度条
- 建造完成：闪光特效+缩放动画
- 生产中：产出动画（每产出一次）
- 升级时：金光+升级特效

**数值反馈**：
- 产出时：浮动数字 "+X 资源名"
- 扣除时：红色浮动数字 "-X 资源名"
- 升级时：属性变化 "+X% 效率提升"

**声音反馈**：
- 建造开始：锤击声
- 建造完成：欢快音效
- 产出时：轻量叮咚声（可关闭）
- 升级时：庆祝音效

## 六、界面设计

### 6.1 建筑面板

```
┌─────────────────────────────────────────────┐
│  建筑列表                                    │
```

## 七、验收标准

### 7.1 功能验收

- [ ] 所有14种建筑可正常建造
- [ ] 建筑建造消耗正确扣除
- [ ] 建筑值班可正常分配/移除角色
- [ ] 建筑产出按正确间隔和数量产出
- [ ] 建筑升级消耗正确扣除
- [ ] 科技解锁条件正确生效
- [ ] 仓库存储上限正确限制

### 7.2 数值验收

- [ ] 各建筑产出效率符合设计
- [ ] 升级消耗符合设计
- [ ] 职业加成正确计算
- [ ] 科技加成正确计算
- [ ] 回报率在合理范围内

### 7.3 体验验收

- [ ] 建造流程清晰直观
- [ ] 升级效果有明显反馈
- [ ] 产出有实时数值显示
- [ ] 异常情况有友好提示
- [ ] 新手引导流畅易懂

### 7.4 性能验收

- [ ] 支持同时运行20+建筑
- [ ] 建造/升级动画流畅
- [ ] 存档/读档正确保存建筑状态
- [ ] 无内存泄漏
